## 分支 1：链表是什么？

```
链表是什么
    │
    ├─ 定义
    │   ├─ 节点组成
    │   ├─ 数据 + 指针
    │   └─ 非连续存储
    │
    ├─ 生活例子
    │   ├─ 寻宝游戏
    │   ├─ 串珠子
    │   └─ 接力赛
    │
    └─ 代码表示
        ├─ class ListNode
        ├─ val（数据）
        └─ next（指针）
```

### 详细解释

**定义分支：**
- 节点：链表的基本单位
- 数据：存储实际的值
- 指针：指向下一个节点的位置
- 非连续：节点分散在内存各处，靠指针连接

**生活例子分支：**
```
寻宝游戏：
宝藏 1 → 纸条（线索）→ 宝藏 2 → 纸条 → 宝藏 3...
每个宝藏都知道下一个在哪！

串珠子：
○—○—○—○—○
每颗珠子都连着下一颗
```

## 分支 2：链表的特点

```
链表的特点
    │
    ├─ 优点 ✓
    │   ├─ 插入删除快 O(1)
    │   ├─ 动态扩容
    │   └─ 不浪费空间
    │
    └─ 缺点 ✗
        ├─ 访问慢 O(n)
        ├─ 占用额外空间（指针）
        └─ 不支持随机访问
```

### 详细解释

**为什么插入删除快？**
```
在位置 i 插入元素 X：

原链表：A -> B -> C
              ↑
           在这里插入 X

步骤：
1. 创建节点 X
2. X.next = C（先连后面）
3. B.next = X（再连前面）

完成！只修改了 2 个指针
不需要移动其他元素 ⚡
```

**为什么访问慢？**
```
要找第 5 个元素：

head -> 1 -> 2 -> 3 -> 4 -> 5 -> ...
        ↑              ↑
       起点          目标

必须一步一步走：
1 → 2 → 3 → 4 → 5

平均要走 n/2 步 🐌
```

## 分支 3：基本操作

```
基本操作
    │
    ├─ 访问 Access
    │   ├─ 从头遍历
    │   └─ O(n)
    │
    ├─ 搜索 Search
    │   ├─ 遍历查找
    │   └─ O(n)
    │
    ├─ 插入 Insert
    │   ├─ 头部插入 O(1)
    │   ├─ 尾部插入 O(n)*
    │   └─ 中间插入 O(1)*
    │
    └─ 删除 Delete
        ├─ 头部删除 O(1)
        ├─ 尾部删除 O(n)*
        └─ 中间删除 O(1)*
        
*需要找到位置，所以是 O(n)
```

### 详细解释

**头部插入（最快）：**
```javascript
// 在头部插入值为 0 的节点
const newNode = new ListNode(0);
newNode.next = head;
head = newNode;

// 完成！O(1)
```

**遍历查找（最常用）：**
```javascript
let current = head;
while (current !== null) {
    console.log(current.val);
    current = current.next;
}
```

## 分支 4：链表类型

```
链表类型
    │
    ├─ 单向链表
    │   └─ 只能往后走
    │
    ├─ 双向链表
    │   ├─ prev（前驱）
    │   └─ next（后继）
    │
    └─ 循环链表
        └─ 尾连头，成环
```

### 详细解释

**单向链表：**
```
head -> ○ -> ○ -> ○ -> null
只能往前走，不能回头
```

**双向链表：**
```
null <- ○ <-> ○ <-> ○ -> null
     ↖___↙ ↖___↙ ↖___↙
    prev  next
可以前后移动
```

**循环链表：**
```
    ┌──────────────┐
    ↓              │
head -> ○ -> ○ -> ○
              ↑______│
尾节点的 next 指向头节点
可以从任意节点开始遍历
```

## 分支 5：时间复杂度对比

```
时间复杂度
    │
    ├─ O(1) ⚡ 超快
    │   ├─ 头部插入
    │   ├─ 头部删除
    │   └─ 已知位置的插入删除
    │
    └─ O(n) 🏃 一般
        ├─ 访问任意元素
        ├─ 搜索元素
        └─ 尾部操作（无尾指针时）
```

### 形象理解

```
O(1) - 开门见山：
就在门口，伸手就到

O(n) - 挨个敲门：
不知道在第几家，要一家家找
```

## 🔑 核心要点总结

### 三个关键词

```
1️⃣ 节点
   链表的基本单位，包含数据和指针

2️⃣ 指针
   连接节点的"线"，指引方向

3️⃣ 灵活
   插拔方便，但访问麻烦
```

### 一个中心思想

```
💡 数据结构的选择是权衡利弊
链表适合频繁插入删除的场景
```

### 最重要的收获

```
✓ 链表是非连续存储
✓ 插入删除只需修改指针
✓ 访问需要遍历
✓ 有单向、双向、循环三种类型
```
