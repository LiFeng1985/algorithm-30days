## 分支 1：基本原理

```
原理
    │
    ├─ 基本思想
    │   ├─ 已排序 + 未排序
    │   └─ 逐个插入到正确位置
    │
    ├─ 生活例子
    │   └─ 整理扑克牌
    │
    └─ 执行过程
        ├─ 第一个元素默认有序
        ├─ 后续元素依次插入
        └─ 每轮保证前面有序
```

---

## 分支 2：代码实现

```
实现
    │
    ├─ 基础版本
    │   ├─ 外层循环遍历元素
    │   ├─ 内层 while 找位置
    │   └─ 移动元素并插入
    │
    ├─ 关键步骤
    │   ├─ 保存当前元素
    │   ├─ 从后向前比较
    │   └─ 找到位置插入
    │
    └─ 注意事项
        ├─ 外层从 i=1 开始
        └─ while 条件用 &&
```

---

## 分支 3：优化方法

```
优化
    │
    ├─ 二分插入排序
    │   ├─ 二分查找插入位置
    │   └─ 减少比较次数
    │
    ├─ 希尔排序
    │   ├─ 分组插入排序
    │   └─ 逐步缩小间隔
    │
    └─ 性能提升
        ├─ 比较次数 O(n) → O(n log n)
        └─ 移动次数不变
```

---

## 分支 4：复杂度与特性

```
复杂度与特性
    │
    ├─ 时间复杂度
    │   ├─ 最好：O(n) - 已有序
    │   ├─ 平均：O(n²)
    │   └─ 最坏：O(n²) - 完全逆序
    │
    ├─ 空间复杂度
    │   └─ O(1) - 原地排序
    │
    ├─ 稳定性
    │   └─ 稳定排序
    │
    └─ 适用场景
        ├─ 数据量小
        ├─ 基本有序
        └─ 在线排序
```

---

## 🔑 核心要点

### 三个关键词
```
1️⃣ 逐个插入
   每次拿一个新元素插入

2️⃣ 前面有序
   保证已排序部分始终有序

3️⃣ 从后向前
   比较时从后面向前找位置
```

### 一个中心思想
```
💡 增量构建
逐步扩大已排序部分的范围
```
