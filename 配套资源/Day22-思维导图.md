## 分支 2：代码实现

```
实现
    │
    ├─ 迭代版本
    │   ├─ while 循环
    │   │   └─ left <= right
    │   ├─ 计算 mid
    │   │   └─ left + (right-left)/2
    │   └─ 调整边界
    │       ├─ left = mid + 1
    │       └─ right = mid - 1
    │
    └─ 递归版本
        ├─ 基准情况
        │   └─ left > right
        ├─ 递归调用
        │   ├─ 左半部分
        │   └─ 右半部分
        └─ 返回值
            ├─ 找到返回索引
            └─ 没找到返回 -1
```

## 分支 3：特性分析

```
特性
    │
    ├─ 时间复杂度
    │   ├─ 最好：O(1)
    │   ├─ 平均：O(log n)
    │   └─ 最坏：O(log n)
    │
    ├─ 空间复杂度
    │   ├─ 迭代：O(1)
    │   └─ 递归：O(log n)
    │
    ├─ 优点
    │   ├─ 查找速度快
    │   ├─ 实现简单
    │   └─ 空间效率高
    │
    └─ 缺点
        ├─ 需要有序数组
        ├─ 不适合链表
        └─ 插入删除慢
```

## 🔑 核心要点

### 三个关键词
```
1️⃣ 有序
   必须是有序数组

2️⃣ 中间
   每次看中间位置

3️⃣ 一半
   排除一半元素
```

### 一个中心思想
```
💡 分而治之
通过不断缩小范围来快速定位
```

### 重要技巧
```
✓ 防止溢出的 mid 计算
✓ 正确的边界更新
✓ 处理找不到的情况
✓ 掌握变体问题
```

## 💡 学习建议

### 理解重点
```
1. 掌握基本实现
2. 理解为什么是 O(log n)
3. 明白边界条件的处理
4. 知道适用场景
```

### 练习方向
```
✓ 能手写迭代版本
✓ 能手写递归版本
✓ 能解决变体问题
✓ 能分析复杂度
```

### 常见误区
```
✗ mid 计算溢出
✗ 边界更新错误
✗ 循环条件用错
✗ 忘记处理找不到的情况
```
