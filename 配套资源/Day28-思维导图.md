## 🔑 核心要点

### 三个关键词
```
1️⃣ 选择
   在当前状态下做出选择

2️⃣ 递归
   基于新状态继续搜索

3️⃣ 撤销
   恢复现场，尝试下一个选择
```

### 一个中心思想
```
💡 试错策略
系统地枚举所有可能
遇到死路就回退
```

### 重要技巧
```
✓ 先写基础回溯框架
✓ 再加剪枝优化
✓ 注意撤销操作
✓ 处理重复元素
```

## 💡 学习建议

### 理解重点
```
1. 掌握回溯的基本框架
2. 理解为什么要撤销选择
3. 学会添加剪枝优化
4. 明白时间复杂度为何高
```

### 练习方向
```
✓ 能手写全排列回溯
✓ 能解决 N 皇后问题
✓ 能实现子集生成
✓ 能添加合适的剪枝
```

### 常见误区
```
✗ 忘记撤销选择
✗ 没有终止条件
✗ 剪枝条件错误
✗ 忽略重复解
```

## 📊 回溯问题识别指南

```
遇到一个问题，如何判断能否用回溯？

1. 是否需要枚举所有可能？
   ↓ 是 → 可能用回溯

2. 是否有明确的约束条件？
   ↓ 是 → 可能用回溯

3. 是否可以逐步构建解？
   ↓ 是 → 可能用回溯

4. 数据规模不大？
   （n <= 20 左右）
   ↓ 是 → 可以用回溯！

经典回溯问题：
✓ 排列组合
✓ N 皇后
✓ 数独
✓ 子集
✓ 括号生成
✓ 单词搜索
✓ 分割回文串
```
