## 分支 1：数组是什么？

```
数组是什么
    │
    ├─ 定义
    │   ├─ 相同类型元素
    │   ├─ 有序集合
    │   └─ 连续内存空间
    │
    ├─ 生活例子
    │   ├─ 一排储物柜
    │   ├─ 电影院座位
    │   └─ 排队做核酸
    │
    └─ 代码表示
        ├─ const arr = [1, 2, 3]
        ├─ new Array(5)
        └─ JavaScript 中的数组
```

### 详细解释

**定义分支：**
- 相同类型：所有元素都是同一类型（数字、字符串等）
- 有序：每个元素有确定的位置（索引）
- 连续：在内存中挨着存放，不分散

**生活例子分支：**
```
储物柜比喻：
┌───┬───┬───┬───┬───┐
│ 0 │ 1 │ 2 │ 3 │ 4 │  ← 编号（索引）
├───┼───┼───┼───┼───┤
│ A │ B │ C │ D │ E │  ← 存储的物品
└───┴───┴───┴───┴───┘

要找 2 号柜的东西，直接走到 2 号位置就行！
```

## 分支 2：数组的特点

```
数组的特点
    │
    ├─ 优点 ✓
    │   ├─ 随机访问快 O(1)
    │   ├─ 占用空间小
    │   └─ 实现简单
    │
    └─ 缺点 ✗
        ├─ 插入删除慢 O(n)
        ├─ 扩容麻烦
        └─ 浪费空间（预分配）
```

### 详细解释

**为什么访问快？**
```
内存地址计算：
地址 = 起始地址 + 索引 × 单个元素大小

例如：
起始地址：1000
索引：3
元素大小：4 字节

地址 = 1000 + 3×4 = 1012

直接跳到 1012 这个位置，一次搞定！⚡
```

**为什么插入删除慢？**
```
在索引 2 处插入元素 X：

原数组：[A, B, C, D, E]
              ↑
              在这里插入 X

步骤：
1. E 往后挪 → [A, B, C, D, _, E]
2. D 往后挪 → [A, B, C, _, D, E]
3. 放入 X   → [A, B, C, X, D, E]

要移动后面所有元素，平均 n/2 次操作！🐌
```

## 分支 3：基本操作

```
基本操作
    │
    ├─ 访问 Access
    │   ├─ arr[index]
    │   └─ O(1) ⚡
    │
    ├─ 搜索 Search
    │   ├─ 遍历查找
    │   └─ O(n)
    │
    ├─ 插入 Insert
    │   ├─ push() 末尾添加
    │   ├─ unshift() 头部添加
    │   └─ O(n) 🐌
    │
    └─ 删除 Delete
        ├─ pop() 末尾删除
        ├─ shift() 头部删除
        └─ O(n) 🐌
```

### 详细解释

**访问操作：**
```javascript
const arr = ['A', 'B', 'C', 'D'];

// 直接访问
console.log(arr[0]);  // 'A' ✅ 快
console.log(arr[2]);  // 'C' ✅ 快
console.log(arr[3]);  // 'D' ✅ 快

不管访问哪个，都是一瞬间！
```

**插入操作对比：**
```javascript
// 末尾插入（快）
arr.push('E');  // O(1) ✅
// 不需要移动其他元素

// 中间插入（慢）
arr.splice(2, 0, 'X');  // O(n) ❌
// 要把后面的元素都往后挪
```

## 分支 4：时间复杂度总结

```
时间复杂度
    │
    ├─ O(1) ⚡ 超快
    │   ├─ 访问任意元素
    │   └─ 末尾增删
    │
    ├─ O(n) 🏃 一般
    │   ├─ 搜索元素
    │   └─ 遍历数组
    │
    └─ O(n) 🐌 慢
        ├─ 中间插入
        └─ 中间删除
```

### 形象理解

```
O(1) - 瞬间移动：
不管数组多大，访问时间都一样
就像坐电梯，按几层都直接到

O(n) - 逐层爬楼：
要找某个元素，可能要一个个看
最坏情况要看遍所有 n 个元素
```

## 分支 5：动态数组

```
动态数组
    │
    ├─ 自动扩容
    │   ├─ 容量不够时
    │   └─ 扩容为 2 倍
    │
    ├─ 扩容过程
    │   ├─ 申请新空间（2 倍大）
    │   ├─ 复制旧数据
    │   └─ 释放旧空间
    │
    └─ 均摊复杂度
        └─ 虽然扩容 O(n)
        └─ 但平均还是 O(1)
```

### 详细解释

**扩容动画：**
```
原始数组：容量 4，已用 4
┌───┬───┬───┬───┐
│ A │ B │ C │ D │
└───┴───┴───┴───┘

添加 E，容量不够！
↓
新数组：容量 8
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ A │ B │ C │ D │ E │   │   │   │
└───┴───┴───┴───┴───┴───┴───┴───┘

复制所有元素，完成扩容！
```

## 🔑 核心要点总结

### 三个关键词

```
1️⃣ 连续
   数组元素在内存中挨着存放

2️⃣ 索引
   通过编号快速访问元素

3️⃣ 权衡
   访问快，但插入删除慢
```

### 一个中心思想

```
💡 数据结构的选择是权衡利弊
没有最好的，只有最合适的
```

### 最重要的收获

```
✓ 数组适合随机访问的场景
✓ 数组不适合频繁插入删除
✓ 动态数组会自动扩容
✓ 理解时间复杂度的含义
```
