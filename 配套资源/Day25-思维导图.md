## 分支 1：基本原理

```
原理
    │
    ├─ 核心思想
    │   └─ 分而治之
    │       ├─ 大问题→小问题
    │       └─ 小问题解→大问题的解
    │
    ├─ 关键特征
    │   ├─ 重叠子问题
    │   │   └─ 相同问题多次出现
    │   └─ 最优子结构
    │       └─ 子问题最优→原问题最优
    │
    ├─ 生活例子
    │   └─ 草稿纸记中间结果
    │
    └─ 为什么快
        └─ 避免重复计算
```

## 分支 2：实现方法

```
方法
    │
    ├─ 记忆化搜索
    │   ├─ 自顶向下
    │   ├─ 递归 + 缓存
    │   ├─ 优点：思路自然
    │   └─ 缺点：递归开销
    │
    ├─ 递推 DP
    │   ├─ 自底向上
    │   ├─ 迭代 + 表格
    │   ├─ 优点：无递归
    │   └─ 缺点：可能算多余
    │
    └─ 空间优化
        ├─ 只保留必要状态
        ├─ O(n) → O(1)
        └─ 技巧：滚动数组
```

## 🔑 核心要点

### 三个关键词
```
1️⃣ 重叠子问题
   相同问题不重复算

2️⃣ 最优子结构
   子问题最优推原问题

3️⃣ 状态转移
   从小推大，从已知推未知
```

### 一个中心思想
```
💡 空间换时间
用额外存储避免重复计算
```

### 重要技巧
```
✓ 先写暴力递归
✓ 再加记忆化
✓ 最后改递推
✓ 能优化就优化
```

## 💡 学习建议

### 理解重点
```
1. 掌握 DP 的基本思想
2. 理解两种实现方法的区别
3. 学会找状态转移方程
4. 明白为什么要避免重复计算
```

### 练习方向
```
✓ 能手写斐波那契的四种解法
✓ 能解决爬楼梯问题
✓ 能分析时间和空间复杂度
✓ 能识别 DP 问题的特征
```

### 常见误区
```
✗ 认为所有问题都能用 DP
✗ 找不到状态转移方程
✗ 忽略边界条件
✗ 空间优化时更新顺序错误
```

## 📊 DP 问题识别指南

```
遇到一个问题，如何判断是否能用 DP？

1. 是否是计数问题？
   （有多少种方法）
   ↓ 是 → 可能用 DP

2. 是否求最值？
   （最大值、最小值、最优解）
   ↓ 是 → 可能用 DP

3. 能否分解为子问题？
   ↓ 是 → 可能用 DP

4. 子问题是否重叠？
   （会重复计算相同的子问题）
   ↓ 是 → 可以用 DP！

5. 是否有最优子结构？
   （子问题的最优解能推出原问题的最优解）
   ↓ 是 → 确定用 DP！
```
