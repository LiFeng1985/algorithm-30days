## 分支 1：树是什么？

```
树是什么
    │
    ├─ 定义
    │   ├─ n 个节点的有限集合
    │   ├─ 层次结构
    │   └─ 一对多关系
    │
    ├─ 生活例子
    │   ├─ 家谱族谱
    │   ├─ 公司架构
    │   └─ 文件系统
    │
    └─ 特点
        ├─ 有且仅有一个根节点
        ├─ 子树不相交
        └─ 每个节点只有一个父节点
```

---

## 分支 2：树的术语

```
树的术语
    │
    ├─ 节点相关
    │   ├─ 根节点（无父节点）
    │   ├─ 叶节点（无子节点）
    │   └─ 内部节点（既有父又有子）
    │
    ├─ 关系相关
    │   ├─ 父节点与子节点
    │   ├─ 兄弟节点（同父）
    │   └─ 祖先与子孙
    │
    └─ 度量相关
        ├─ 度（子节点个数）
        ├─ 深度（层数）
        └─ 高度（最大深度）
```

---

## 分支 3：二叉树

```
二叉树
    │
    ├─ 定义
    │   └─ 每个节点最多 2 个子节点
    │
    ├─ 特点
    │   ├─ 区分左右
    │   └─ 有序性
    │
    └─ 特殊类型
        ├─ 满二叉树
        │   └─ 每层都满
        │
        └─ 完全二叉树
            └─ 最后一层靠左
```

---

## 分支 4：遍历方式

```
遍历方式
    │
    ├─ 前序遍历
    │   └─ 根→左→右
    │
    ├─ 中序遍历
    │   └─ 左→根→右
    │
    ├─ 后序遍历
    │   └─ 左→右→根
    │
    └─ 层序遍历
        └─ 从上到下，从左到右
```

---

## 🔑 核心要点

### 三个关键词
```
1️⃣ 层次
   树是层次结构

2️⃣ 父子关系
   每个节点（除根）都有一个父节点

3️⃣ 递归
   树天然适合递归处理
```

### 一个中心思想
```
💡 从根开始，分而治之
大问题分解成小问题
```
