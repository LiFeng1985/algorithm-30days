### 题目 2：生活中的算法（15 分）

**参考答案：**

**示例 1：刷牙的算法**
```
步骤：
1. 挤牙膏到牙刷上
2. 把牙刷放进嘴里
3. 来回刷牙齿表面
4. 漱口
5. 清洗牙刷
```

**示例 2：去学校的算法**
```
步骤：
1. 从家出发
2. 走到公交站
3. 坐 3 路公交车
4. 在学校站下车
5. 走进校门
```

**示例 3：洗衣服的算法**
```
步骤：
1. 把衣服放进洗衣机
2. 加洗衣液
3. 选择洗涤模式
4. 按启动按钮
5. 洗好后取出晾晒
```

**评分要点：**
- 每个例子 5 分（步骤清晰 3 分，合理 2 分）
- 必须是日常生活中的事情
- 步骤要有明确的顺序
- 不能太复杂（5-8 步为宜）

### 题目 3：好算法的标准（10 分）

**参考答案：**

**好算法的标准：**

1. **正确性**（3 分）
   - 能正确解决问题
   - 结果准确无误

2. **效率高**（3 分）
   - 执行时间短
   - 占用资源少

3. **易理解**（2 分）
   - 逻辑清晰
   - 便于维护

4. **健壮性**（2 分）
   - 能处理异常情况
   - 适应不同输入

**其他可接受答案：**
- 简单明了
- 可读性强
- 通用性好
- 节省空间

**评分要点：**
- 每答对一点得相应分数
- 意思对即可，不要求文字完全一致
- 能举例说明的额外 +2 分

## 二、理解应用题答案

### 题目 4：比较两种方法（15 分）

**答案与解析：**

**1. 哪种方法更快？为什么？（5 分）**

**答案：** 方法 B 更快（2 分）

**原因：** （3 分）
```
方法 A 要翻 250 次
方法 B 只需要翻几次就能找到大概位置
方法 B 利用了书的页码规律
这就是"二分查找"的思想
```

**评分要点：**
- 选对得 2 分
- 提到"次数少"得 2 分
- 提到"利用规律"得 1 分

---

**2. 如果书只有 10 页，哪种方法合适？（5 分）**

**答案：** 方法 A 也可以（2 分）

**原因：** （3 分）
```
因为只有 10 页，一页页翻也很快
不需要复杂的查找方法
简单直接反而更好
```

**评分要点：**
- 选 A 或 B 都可以，关键看理由
- 提到"数据量小"得 2 分
- 提到"简单就好"得 1 分

---

**3. 这说明了什么道理？（5 分）**

**答案要点：**
```
不同的问题要用不同的方法
数据量大时，好算法优势明显
数据量小时，简单方法也可以
要根据实际情况选择算法
```

**评分要点：**
- 答出"不同情况不同方法"得 3 分
- 答出"数据量影响选择"得 2 分

---

### 题目 5：工作效率对比（10 分）

**答案与解析：**

**1. 50 份试卷时（5 分）**

**答案：** 方案 B 更聪明（2 分）

**理由：** （3 分）
```
先分组（分治思想）
每组数量少，排序快

```

**评分要点：**
- 选 B 得 2 分
- 提到"分组"得 1 分
- 提到"分治"得 2 分

---

**2. 5 份试卷时（5 分）**

**答案：** 方案 A 也可以（2 分）

**理由：** （3 分）
```
数量很少，不需要复杂方法
直接比较也很快
过度优化反而麻烦
```

**评分要点：**
- 理解"数据量小不需要优化"即可得分
- 言之有理即可

## 三、编程思维题答案

### 题目 6：泡面的算法（10 分）

**参考答案：**

```
算法名称：热泡面
输入：一碗泡面、微波炉
输出：热好的泡面

步骤：
1. 确认泡面碗可以放进微波炉
2. 打开微波炉门
3. 把泡面碗放在转盘中间
4. 关上微波炉门
5. 设置时间 2 分钟
6. 按启动按钮
7. 听到"叮"声后，小心取出（很烫！）
8. 搅拌均匀，开吃！

注意事项：
- 不能用金属碗或带金边的碗
- 取出时要戴手套或用毛巾
- 不要加热太久，会炸开
- 让大人帮忙更安全
```

**评分要点：**
- 步骤完整（5 分）
- 有安全意识（3 分）
- 考虑周到（2 分）

**扣分项：**
- 缺少关键步骤（如关门）扣 2 分
- 没有安全提醒扣 3 分
- 步骤不合理扣 2 分

### 题目 7：判断闰年（10 分）

**参考答案：**

```
算法：判断闰年
输入：年份 year
输出：true（是闰年）或 false（不是闰年）

步骤：
1. 如果 year 能被 400 整除，返回 true
2. 否则，如果 year 能被 100 整除，返回 false
3. 否则，如果 year 能被 4 整除，返回 true
4. 否则，返回 false

或者用代码表示：
if (year % 400 == 0) return true;
else if (year % 100 == 0) return false;
else if (year % 4 == 0) return true;
else return false;

测试：
2024 年是闰年吗？✓ 是（2024÷4=506，整除）
2023 年是闰年吗？✗ 不是（2023÷4=505...3，不整除）
2000 年是闰年吗？✓ 是（2000÷400=5，整除）
1900 年是闰年吗？✗ 不是（1900÷100=19，但÷400 不整除）
```

**评分要点：**
- 步骤正确（6 分，每一步 1.5 分）
- 测试结果正确（4 分，每个 1 分）
- 能用代码表示额外 +2 分

**常见错误：**
❌ 忘记 400 的特殊规则
❌ 顺序搞错（应该先判断 400）
❌ 测试算错

## 💡 错题分析

### 常见错误类型

**1. 概念理解错误**
```
错误：算法就是电脑程序
纠正：算法是方法，程序是算法的实现
```

**2. 绝对化思维**
```
错误：算法越复杂越好
纠正：简单有效的才是好算法
```

**3. 忽视实际应用**
```
错误：算法只在编程中用
纠正：生活中处处有算法
```

### 改进建议

```
□ 多观察生活中的算法
□ 尝试用算法思维思考问题
□ 多练习，熟能生巧
□ 向他人讲解，加深理解
□ 不要怕错，从错误中学习
```
