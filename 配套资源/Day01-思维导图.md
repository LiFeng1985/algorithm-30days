## 分支 1：算法是什么？

```
算法是什么
    │
    ├─ 定义
    │   ├─ 解决问题的步骤
    │   ├─ 完成指令序列
    │   └─ 方法 + 过程
    │
    ├─ 生活例子
    │   ├─ 菜谱（做菜步骤）
    │   ├─ 导航（路线规划）
    │   ├─ 说明书（使用方法）
    │   └─ 课表（学习安排）
    │
    └─ 程序例子
        ├─ 排序（整理数据）
        ├─ 查找（找东西）
        ├─ 计算（数学运算）
        └─ 搜索（搜索引擎）
```

### 详细解释

**定义分支：**
- 算法不是具体的东西，而是"方法"
- 就像"菜谱"告诉你要先洗菜、再切菜、然后炒菜
- 算法告诉计算机第一步做什么、第二步做什么...

**生活例子分支：**
```
菜谱算法：
输入：食材、调料
处理：洗→切→炒→调味
输出：一道菜

导航算法：
输入：起点 A、终点 B
处理：规划路线、避开拥堵
输出：最佳路径
```

## 分支 3：好算法的特点

```
好算法的特点
    │
    ├─ 正确性 ✓
    │   └─ 能解决问题
    │
    ├─ 效率高 ⚡
    │   ├─ 时间短
    │   └─ 占用少
    │
    ├─ 易理解 📖
    │   ├─ 逻辑清晰
    │   └─ 好维护
    │
    └─ 健壮性 💪
        └─ 能处理异常
```

### 详细解释

**正确性：**
```
✓ 能正确解决问题
✓ 结果准确无误
✗ 错误百出的算法，再快也没用
```

**效率高：**
```
时间对比：
算法 A：处理 1000 个数要 1 秒
算法 B：处理 1000 个数要 0.01 秒
→ 选 B！

空间对比：
算法 A：需要 1GB 内存
算法 B：只需要 1MB 内存
→ 选 B！
```

**易理解：**
```
好代码：
一看就懂
容易修改
方便调试

烂代码：
天书一样
不敢乱动
难以维护
```

## 📝 使用指南

### 如何使用这份思维导图

**第一次学习：**
1. 先看中心主题
2. 顺着分支往下看
3. 每个分支都理解
4. 用自己的话复述

**复习时：**
1. 看着图回忆详细内容
2. 遮住细节，尝试默写
3. 重点标记薄弱点
4. 定期回顾

**教学时：**
1. 用导图作为讲解框架
2. 每个分支展开说明
3. 添加自己的例子
4. 引导学生思考
