## 一、基础概念题答案

### 题目 1：什么是链表？（10 分）

**参考答案：**

**标准答案示例：**
```
链表是由节点组成的数据结构，
每个节点包含数据和指向下一个节点的指针。
节点在内存中分散存储，通过指针连接。
就像寻宝游戏，每个线索告诉你下一个地点在哪。
```

**评分要点：**
- ✅ 提到"节点"（3 分）
- ✅ 提到"指针"或"连接"（3 分）
- ✅ 提到"非连续存储"（2 分）
- ✅ 用自己的话表达（2 分）

**优秀答案示例：**
```
"链表就像一串珠子，
每颗珠子都知道下一颗在哪，
用线（指针）连起来。"
```

### 题目 2：链表 vs 数组（15 分）

**参考答案：**

| 对比项 | 数组 | 链表 |
|--------|------|------|
| **存储方式** | 连续内存 | 分散内存 + 指针 |
| **访问速度** | O(1) 快 | O(n) 慢 |
| **插入删除** | O(n) 慢 | O(1) 快 |
| **空间分配** | 固定或扩容 | 动态分配 |
| **随机访问** | 支持 | 不支持 |

**评分要点：**
- 每个区别 5 分（对比正确 3 分，说明清楚 2 分）
- 至少答出 3 个区别
- 能举例说明额外 +3 分

### 题目 3：时间复杂度分析（10 分）

**参考答案：**

**访问慢的原因：**（5 分）
```
链表不是连续存储的，
不能通过计算直接跳到某个位置。
要找第 n 个节点，必须从头开始，
一个一个往下找，平均要找 n/2 次。
所以时间复杂度是 O(n)。
```

**插入删除快的原因：**（5 分）
```
在已知位置插入或删除时，
只需要修改相邻节点的指针，
不需要移动其他元素。
就像穿珠子一样，拔掉一颗再穿上一颗，
其他珠子不用动。
所以时间复杂度是 O(1)。
```

**评分要点：**
- 提到"不能直接跳转"得 2 分
- 提到"需要遍历"得 2 分
- 提到"修改指针"得 3 分
- 有比喻说明额外 +3 分

## 二、代码实践题答案

### 题目 4：定义链表节点（15 分）

**参考答案：**

```javascript
class ListNode {
    constructor(val) {
        this.val = val;      // 存储的值
        this.next = null;     // 指向下一个节点的指针，初始为 null
    }
}

// 测试
const node1 = new ListNode(1);
const node2 = new ListNode(2);
const node3 = new ListNode(3);

console.log(node1);  // ListNode { val: 1, next: null }
```

**评分要点：**
- `this.val = val`（5 分）
- `this.next = null`（5 分）
- 能通过测试（5 分）

**常见错误：**
❌ `this.next = undefined` → ✅ 应该初始化为 `null`
❌ 忘记写 `this.` → ✅ 实例属性要用 `this`

### 题目 5：创建链表（20 分）

**参考答案：**

```javascript
// 1. 创建 5 个节点
const n1 = new ListNode(1);
const n2 = new ListNode(2);
const n3 = new ListNode(3);
const n4 = new ListNode(4);
const n5 = new ListNode(5);

// 2. 连接节点
n1.next = n2;
n2.next = n3;
n3.next = n4;
n4.next = n5;

// 3. 遍历链表
let current = n1;  // 或 head

    console.log(current.val);
    current = current.next;
}
// 输出：1 2 3 4 5
```

**评分要点：**

**常见错误：**
❌ `while (current)` → ✅ 最好写 `current !== null`
❌ 忘记 `current = current.next` → ✅ 会死循环

### 题目 6：在头部插入节点（15 分）

**参考答案：**

```javascript
function insertAtHead(head, val) {
    const newNode = new ListNode(val);  // 创建新节点
    newNode.next = head;  // 新节点指向原头节点
    return newNode;  // 返回新头节点
}

// 测试
// 原链表：1 -> 2 -> 3
const newHead = insertAtHead(head, 0);
// 新链表：0 -> 1 -> 2 -> 3
```

**评分要点：**
- 创建新节点（5 分）
- 正确设置 next 指针（5 分）
- 返回新的头节点（5 分）

**图解：**
```
原链表：head -> 1 -> 2 -> 3

插入 0：
步骤 1：创建节点 0
步骤 2：让 0 的 next 指向 1
步骤 3：返回 0 作为新 head

新链表：newHead -> 0 -> 1 -> 2 -> 3
```

### 题目 7：查找元素（15 分）

**参考答案：**

```javascript
function findInList(head, target) {
    let current = head;
    let index = 0;
    
    while (current !== null) {
        if (current.val === target) {
            return index;  // 找到了，返回索引
        }
        current = current.next;
        index++;
    }
    
    return -1;  // 没找到
}

// 测试
// 链表：1 -> 2 -> 3 -> 4 -> 5
console.log(findInList(head, 3));  // 2
console.log(findInList(head, 6));  // -1
```

**评分要点：**
- 正确遍历链表（5 分）
- 正确比较值（5 分）
- 正确处理找到和没找到的情况（5 分）

```
- 每首歌是一个节点

- 可以随时添加/删除歌曲
```

**其他可接受答案：**
- 撤销/重做功能

**参考答案：**

## 四、拓展挑战题答案

### 题目 10：反转链表（15 分）

**参考答案：**

**方法 1：迭代法（推荐）**
```javascript
function reverseList(head) {
    let prev = null;      // 前一个节点
    let current = head;   // 当前节点
    
    while (current !== null) {
        const nextTemp = current.next;  // 先保存下一个节点
        current.next = prev;            // 反转指针
        prev = current;                 // prev 前进一步
        current = nextTemp;             // current 前进一步
    }
    
    return prev;  // prev 就是新的头节点
}
```

**图解过程：**
```
原链表：1 -> 2 -> 3 -> null

第一步：
prev=null, current=1
反转后：null <- 1    2 -> 3

第二步：
prev=1, current=2
反转后：null <- 1 <- 2    3

第三步：
prev=2, current=3
反转后：null <- 1 <- 2 <- 3

结束！返回 prev（节点 3）
新链表：3 -> 2 -> 1 -> null
```

**评分要点：**
- 三个指针使用正确（5 分）
- 循环逻辑正确（5 分）
- 返回结果正确（5 分）

### 题目 11：检测环形链表（15 分）

**参考答案：**

**快慢指针法（Floyd 判圈算法）**
```javascript
function hasCycle(head) {
    if (head === null || head.next === null) {
        return false;
    }
    
    let slow = head;          // 慢指针，每次走一步
    let fast = head.next;     // 快指针，每次走两步
    
    while (fast !== null && fast.next !== null) {
        if (slow === fast) {
            return true;  // 相遇说明有环
        }
        slow = slow.next;
        fast = fast.next.next;
    }
    
    return false;  // 走到尽头，没有环
}
```

**原理解释：**
```
想象两个人在操场上跑步：
- 慢的每秒跑 1 米
- 快的每秒跑 2 米

如果操场是直的：
快的先到终点，不会相遇

如果操场是环形的：
快的会追上慢的，两人相遇

链表也是这个道理！
```

**评分要点：**
- 边界条件处理（3 分）
- 快慢指针设置（4 分）
- 循环条件正确（4 分）
- 判断逻辑正确（4 分）

## 💡 错题分析

### 常见错误类型

**1. 指针操作错误**
```
错误：搞不清指针指向哪里
纠正：画图！画图！画图！
练习：用纸笔画出每个节点的连接关系
```

**2. 边界条件遗漏**
```
错误：忘记检查空链表
纠正：养成习惯，先检查 head 是否为 null
```

**3. 指针丢失**
```
错误：操作过程中弄丢了指针
纠正：先保存再修改，注意顺序
记忆：断链前先搭桥
```
