## 一、基础概念题答案

### 题目 1：什么是贪心算法？（10 分）

**参考答案：**
```
贪心算法是一种最优化算法，
在每一步都选择当前看起来最好的方案，
希望通过局部最优达到全局最优。
就像买东西只选性价比最高的一样。
```

**评分要点：**
- ✅ 提到"每一步选择最好"（4 分）
- ✅ 有自己的比喻（3 分）
- ✅ 提到"局部最优"或"当前最好"（3 分）

### 题目 2：贪心 vs 动态规划（15 分）

**参考答案：**

**相同点：**（3 分）
```
1. 都用于最优化问题
2. 都具有最优子结构
3. 都可以分解为子问题
```

**不同点：**（10 分）

**1. 最优解保证：**（2 分）
```
贪心：不一定得到全局最优
DP: 一定能得到全局最优
```

**2. 决策方式：**（3 分）
```
贪心：自顶向下，每次做贪心选择
DP: 自底向上，先解决所有子问题
```

**3. 时间复杂度：**（2 分）
```
贪心：通常 O(n) 或 O(n log n)，较快
DP: 通常 O(n²) 或更高，较慢
```

**4. 适用场景：**（3 分）
```
贪心适合：
- 有贪心选择性质的问题
- 对时间要求高的场景
- 近似解可接受的情况

DP 适合：
- 需要精确最优解
- 子问题重叠明显
- 贪心不适用的情况
```

**你会选择哪种方法：**（额外加分）
```
优先尝试贪心（简单高效），
如果不行再用 DP（保证最优）。
```

**评分要点：**
- 相同点 3 分
- 不同点每项按分值给分

### 题目 3：贪心的局限性（10 分）

**参考答案：**

**为什么贪心不一定得到最优解：**（3 分）
```
因为贪心只看眼前，不考虑长远影响。
当前的最优选择可能导致后续的选择变差，
最终结果不是全局最优。
```

**举一个贪心失败的例子：**（3 分）
```
找零钱问题，币值为 [25, 20, 5, 1]，要找 40：
贪心：25 + 5 + 5 + 5 = 4 个硬币
最优：20 + 20 = 2 个硬币

0-1 背包问题：
贪心按单位价值选，可能装不满或装不下大的，
DP 能保证最优。
```

**如何判断一个问题能否用贪心：**（2 分）
```
1. 证明贪心选择性质：
   存在某个最优解包含贪心选择
   
2. 证明最优子结构：
   做出贪心选择后，剩余子问题的最优解
   与原问题的最优解组合仍是最优

3. 数学归纳法或反证法
```

**贪心算法的优势：**（2 分）
```
✓ 实现简单
✓ 时间复杂度低
✓ 空间效率高
✓ 容易理解和调试
```

**评分要点：**
- 原因说明 3 分
- 举例 3 分
- 判断方法 2 分
- 优势 2 分

## 二、代码实践题答案

### 题目 4：活动选择问题（25 分）

**参考答案：**

```javascript
/**
 * 活动选择问题
 */
function activitySelector(startTimes, endTimes) {
    const n = startTimes.length;
    
    // 创建活动对象并排序
    const activities = [];
    for (let i = 0; i < n; i++) {
        activities.push({
            index: i,
            start: startTimes[i],
            end: endTimes[i]
        });
    }
    
    // 按结束时间升序排序
    activities.sort((a, b) => a.end - b.end);
    
    const selected = [];
    let lastEnd = -Infinity;
    
    for (const activity of activities) {
        if (activity.start >= lastEnd) {
            // 不冲突，可以选择
            selected.push(activity.index);
            lastEnd = activity.end;
        }
    }
    
    return selected;
}

// 测试
const starts1 = [1, 3, 0, 5, 8, 5];
const ends1 = [2, 4, 6, 7, 9, 9];

console.log('选中的活动:', activitySelector(starts1, ends1));
// 可能输出：[0, 1, 3, 4] ✓

const starts2 = [1, 2, 3, 4, 5];
const ends2 = [2, 3, 4, 5, 6];

console.log('最多能参加的活动数:', activitySelector(starts2, ends2).length);
// 5 ✓
```

**贪心策略的正确性证明思路：**
```
要证明：按结束时间排序的贪心策略能得到最优解

证明思路（反证法）：
1. 假设贪心解 G 不是最优解，存在更优解 O
2. 比较 G 和 O，找到第一个不同的活动
3. 由于 G 选择的是结束最早的，所以 G 的这个活动结束时间 <= O 的对应活动
4. 因此可以用 G 的活动替换 O 的活动，仍然保持可行性
5. 重复这个过程，最终 O 可以变成 G，矛盾！
6. 所以 G 就是最优解
```

**评分要点：**
- 排序正确 6 分
- 贪心选择正确 8 分
- 边界处理正确 4 分
- 返回值正确 2 分
- 能通过测试 3 分
- 理解正确性证明 2 分（加分）

**常见错误：**
❌ 没有排序 → ✅ 必须按结束时间排序
❌ 比较条件错误 → ✅ 是当前开始时间 >= 上次结束时间
❌ 忘记保存索引 → ✅ 如果需要返回原索引

### 题目 5：找零钱问题（20 分）

**参考答案：**

```javascript
/**
 * 找零钱问题（贪心版本）
 */
function makeChange(coins, amount) {
    const result = {};
    
    for (const coin of coins) {
        result[coin] = 0;
    }
    
    let remaining = amount;
    
    for (const coin of coins) {
        if (remaining <= 0) break;
        
        const count = Math.floor(remaining / coin);
        
        if (count > 0) {
            result[coin] = count;
            remaining -= coin * count;
        }
    }
    
    return result;
}

// 测试
console.log(makeChange([25, 10, 5, 1], 63));
// {25: 2, 10: 1, 5: 0, 1: 3} ✓
// 验证：2*25 + 1*10 + 0*5 + 3*1 = 63

console.log(makeChange([25, 10, 5, 1], 40));
// {25: 1, 10: 1, 5: 1, 1: 0} ✓
// 验证：1*25 + 1*10 + 1*5 = 40

// 贪心不是最优的例子
console.log(makeChange([25, 20, 5, 1], 40));
// {25: 1, 20: 0, 5: 3, 1: 0} - 4 个硬币
// 但最优解是：{25: 0, 20: 2, 5: 0, 1: 0} - 2 个硬币
```

**为什么有些币值系统贪心不是最优：**
```
关键：币值之间是否有整除关系

标准币值 [25, 10, 5, 1]：
- 25 = 2*10 + 5
- 10 = 2*5
- 5 = 5*1
这种有整除关系的，贪心是最优的

非标准币值 [25, 20, 5, 1]：
- 25 和 20 没有好的整除关系
- 导致贪心可能不是最优
```

**评分要点：**
- 贪心逻辑正确 10 分
- 计数正确 5 分
- 返回值正确 3 分
- 能通过测试 2 分
- 理解局限性 额外加分

### 题目 6：分数背包问题（15 分）

**参考答案：**

```javascript
/**
 * 分数背包问题
 */
function fractionalKnapsack(items, capacity) {
    // 计算单位价值
    items.forEach(item => {
        item.unitValue = item.value / item.weight;
    });
    
    // 按单位价值降序排序
    items.sort((a, b) => b.unitValue - a.unitValue);
    
    let totalValue = 0;
    let remaining = capacity;
    
    for (const item of items) {
        if (remaining <= 0) break;
        
        if (item.weight <= remaining) {
            // 可以全部装入
            totalValue += item.value;
            remaining -= item.weight;
        } else {
            // 只能装一部分
            const fraction = remaining / item.weight;
            totalValue += item.value * fraction;
            remaining = 0;
        }
    }
    
    return totalValue;
}

// 测试
const items1 = [
    { weight: 10, value: 60 },
    { weight: 20, value: 100 },
    { weight: 30, value: 120 }
];

console.log('最大价值:', fractionalKnapsack(items1, 50));
// 240 ✓
// 解释：
// 物品 1：单位价值 6，全装，价值 60，剩余容量 40
// 物品 2：单位价值 5，全装，价值 100，剩余容量 20
// 物品 3：单位价值 4，装 2/3，价值 80，剩余容量 0
// 总计：60 + 100 + 80 = 240

const items2 = [
    { weight: 5, value: 30 },
    { weight: 10, value: 40 },
    { weight: 15, value: 45 }
];

console.log('最大价值:', fractionalKnapsack(items2, 20));
// 95 ✓
// 物品 1：单位价值 6，全装，价值 30，剩余容量 15
// 物品 2：单位价值 4，全装，价值 40，剩余容量 5
// 物品 3：单位价值 3，装 1/3，价值 15，剩余容量 0
// 总计：30 + 40 + 15 = 95
```

**为什么分数背包可以用贪心：**
```
因为可以分割物品！

策略：
1. 总是优先装单位价值最高的
2. 如果装不下完整的，就装一部分
3. 这样保证每单位容量都产生最大价值

证明思路：
假设最优解没有装单位价值最高的物品 i
那么可以用 i 替换其他物品的一部分
总价值会增加，矛盾！
所以贪心就是最优
```

**评分要点：**
- 单位价值计算正确 3 分
- 排序正确 3 分
- 贪心逻辑正确 5 分
- 分数计算正确 2 分
- 返回值正确 1 分
- 能通过测试 1 分

### 题目 8：算法对比总结（10 分）

**参考答案：**

**1. 暴力枚举：**（2 分）
```
时间复杂度：O(2^n) 或 O(n!)
优点：保证最优，实现简单
缺点：效率极低，只适合小规模
```

**2. 动态规划：**（2 分）
```
时间复杂度：O(n²) 或 O(n³)
优点：保证最优，适用面广
缺点：复杂度高，需要较多空间
```

**3. 贪心算法：**（2 分）
```
时间复杂度：O(n) 或 O(n log n)
优点：简单高效，空间省
缺点：不一定最优，适用范围窄
```

**选择策略：**（2 分）
```
- 数据规模小：暴力枚举（简单直接）
- 要求最优解：动态规划（保证最优）
- 贪心能用：贪心算法（高效）
- 时间紧迫：贪心或近似算法
```

**总结：**（2 分）
```
1. 先判断问题类型（最优化/计数/存在性）
2. 分析数据规模和性能要求
3. 优先考虑贪心（如果能用）
4. 贪心不行用 DP
5. 规模很小可以用暴力
6. 实际问题可能需要启发式或近似
```

**评分要点：**
- 三种算法各 2 分
- 选择策略 2 分
- 总结合理 2 分

```
大家好，今天我要讲的是"贪心算法"。

贪心不一定总是最好的。

贪心会给你：25 + 5 + 5 + 5 = 4 个硬币
但最好的是：20 + 20 = 2 个硬币

所以，贪心算法就是：
每次都选眼前最好的，
希望能达到整体最优。
有时候很有效，
但不保证总是最优！

谢谢大家！
```
