### 题目 2：数组的特点（15 分）

**参考答案：**

**主要特点：**

1. **随机访问快**（5 分）
   - 可以通过索引直接访问任意元素
   - 时间复杂度 O(1)
   - 因为内存是连续的，可以计算位置

2. **插入删除慢**（5 分）
   - 在中间插入或删除需要移动其他元素
   - 时间复杂度 O(n)
   - 因为要保持连续性

3. **占用连续内存**（3 分）
   - 所有元素在内存中挨着存放
   - 这是访问快的原因
   - 也是扩容麻烦的原因

4. **长度固定**（2 分）
   - 静态数组长度不可变
   - 动态数组扩容需要重新分配内存

**评分要点：**
- 每答对一点得相应分数
- 意思对即可
- 能举例说明额外 +2 分

### 题目 3：时间复杂度（10 分）

**参考答案：**

**访问快的原因：**（5 分）
```
数组在内存中是连续存储的。
知道第一个元素的地址后，
可以通过计算直接跳到任意位置：
地址 = 起始地址 + 索引 × 单个元素大小

所以不管访问哪个元素，
都只需要一次计算，时间 O(1)。
```

**插入删除慢的原因：**（5 分）
```
在数组中间插入元素时，
后面的元素都要往后挪一位。
删除时，后面的元素都要往前挪。

平均要移动 n/2 个元素，
所以时间复杂度是 O(n)。
```

**评分要点：**
- 提到"连续存储"得 2 分
- 提到"计算地址"得 2 分
- 提到"移动元素"得 3 分
- 有具体例子额外 +3 分

## 二、代码实践题答案

### 题目 4：数组基本操作（20 分）

**参考答案：**

```javascript
// 1. 创建一个包含 [1, 2, 3, 4, 5] 的数组
const arr = [1, 2, 3, 4, 5];

// 2. 访问第 3 个元素（索引为 2）
const third = arr[2];
console.log(third); // 输出 3

// 3. 在数组末尾添加数字 6
arr.push(6);
console.log(arr); // [1, 2, 3, 4, 5, 6]

// 4. 删除最后一个元素
arr.pop();
console.log(arr); // [1, 2, 3, 4, 5]

// 5. 计算数组长度
const len = arr.length;
console.log(len); // 5
```

**评分要点：**
- 每空 4 分
- `push()`和 `pop()` 必须写对
- 索引访问用 `[]`
- `length` 属性正确使用

**常见错误：**
❌ `arr[3]` → ✅ 应该是 `arr[2]`（索引从 0 开始）
❌ `arr.add(6)` → ✅ 应该是 `arr.push(6)`
❌ `arr.size` → ✅ 应该是 `arr.length`

### 题目 5：遍历数组（15 分）

**参考答案：**

```javascript
const numbers = [10, 20, 30, 40, 50];

// 方法 1：for 循环
for (let i = 0; i < numbers.length; i++) {
    console.log(numbers[i]);
}

// 方法 2：forEach
numbers.forEach((num, index) => {
    console.log(num);
});

// 方法 3：for...of
for (const num of numbers) {
    console.log(num);
}
```

**评分要点：**
- 每种方法 5 分
- 语法正确 3 分
- 能正确输出 2 分

**其他可接受方法：**
```javascript
// 方法 4：for...in（不推荐，但可以）
for (let index in numbers) {
    console.log(numbers[index]);
}

// 方法 5：map（会返回新数组）
numbers.map(num => console.log(num));
```

### 题目 6：数组查找（15 分）

**参考答案：**

```javascript
function findIndex(arr, target) {
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] === target) {
            return i;
        }
    }
    return -1;
}

// 测试
console.log(findIndex([1, 3, 5, 7, 9], 5));  // 2
console.log(findIndex([1, 3, 5, 7, 9], 6));  // -1
```

**评分要点：**
- 循环遍历数组（5 分）
- 正确比较（5 分）
- 找到返回索引，没找到返回 -1（5 分）

**优化版本（额外加分）：**
```javascript
// 使用内置方法
function findIndex(arr, target) {
    return arr.indexOf(target);
}
```

### 题目 8：数组 vs 其他结构（10 分）

**参考答案：**

**我的选择：** 不会选择数组（2 分）

**理由：**（8 分）
```
因为需要在中间频繁插入删除，
数组每次操作都要移动大量元素，
效率太低（O(n)）。

应该选择链表，
链表插入删除只需要修改指针，
不需要移动元素，效率更高（O(1)）。
```

**评分要点：**
- 选"不会"得 2 分
- 提到"移动元素"得 3 分
- 提到"效率低"或"O(n)"得 2 分
- 提出替代方案（如链表）得 3 分

**如果选择数组，理由合理也可得分：**
```
如果数据量很小，
或者主要是访问操作，
数组也可以用。
```

## 四、拓展挑战题答案

### 题目 9：数组去重（10 分）

**参考答案：**

**方法 1：使用 Set**
```javascript
function removeDuplicates(arr) {
    return [...new Set(arr)];
}
```

**方法 2：使用 filter**
```javascript
function removeDuplicates(arr) {
    return arr.filter((item, index) => 
        arr.indexOf(item) === index
    );
}
```

**方法 3：使用循环**
```javascript
function removeDuplicates(arr) {
    const result = [];
    for (let i = 0; i < arr.length; i++) {
        if (!result.includes(arr[i])) {
            result.push(arr[i]);
        }
    }
    return result;
}
```

**评分要点：**
- Set 方法：5 分（最简洁）
- filter 方法：7 分（巧妙）
- 循环方法：10 分（容易理解）
- 能通过测试用例：额外 +5 分

### 题目 10：两数之和（10 分）

**参考答案：**

**方法 1：暴力解法**
```javascript
function twoSum(nums, target) {
    for (let i = 0; i < nums.length; i++) {
        for (let j = i + 1; j < nums.length; j++) {
            if (nums[i] + nums[j] === target) {
                return [i, j];
            }
        }
    }
    return [];
}
```

**方法 2：哈希表优化（推荐）**
```javascript
function twoSum(nums, target) {
    const map = new Map();
    
    for (let i = 0; i < nums.length; i++) {
        const complement = target - nums[i];
        
        if (map.has(complement)) {
            return [map.get(complement), i];
        }
        
        map.set(nums[i], i);
    }
    
    return [];
}
```

**评分要点：**
- 暴力解法：6 分（思路正确）
- 哈希表解法：10 分（最优解）
- 能通过测试：额外 +5 分
- 时间复杂度分析：额外 +5 分

**复杂度对比：**
```
暴力法：O(n²) - 两层循环
哈希表：O(n) - 一层循环，Map 查找 O(1)
```

## 💡 错题分析

### 常见错误类型

**1. 索引错误**
```
错误：认为索引从 1 开始
纠正：JavaScript 索引从 0 开始
练习：多写几次 arr[0] 访问第一个元素
```

**2. 方法混淆**
```
错误：add/remove 当成 push/pop
纠正：数组添加用 push()，删除用 pop()
记忆：push 是推入，pop 是弹出
```

**3. 不理解复杂度**
```
错误：觉得数组插入很快
纠正：数组插入需要移动元素，所以慢
对比：访问快（直接计算），插入慢（要移动）
```
