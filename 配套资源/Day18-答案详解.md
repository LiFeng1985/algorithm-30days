## 一、基础概念题答案

### 题目 1：什么是归并排序？（10 分）

**参考答案：**
```
归并排序是一种高效的排序算法，
采用分治法的思想，
先将数组分解成小问题，
分别解决后再合并起来。
就像整理两堆有序的牌。
```

- ✅ 提到"分治法"或"分解合并"（4 分）

### 题目 2：分治法的思想（15 分）

**参考答案：**

**分治法的三个步骤：**（9 分）

**1. 分解（Divide）：**（3 分）
```
将原问题分解为若干个规模较小的子问题。
在归并排序中，就是把数组从中间分成两半。
```

**2. 解决（Conquer）：**（3 分）
```
递归地解决这些子问题。
如果子问题足够小（只有一个元素），直接返回。
```

**3. 合并（Combine）：**（3 分）
```
将子问题的解合并起来，得到原问题的解。
在归并排序中，就是合并两个有序的子数组。
```

**为什么分治法有效：**（6 分）
```
1. 化繁为简：把大问题拆成小问题，更容易解决
2. 并行处理：子问题可以独立解决，理论上可以并行
3. 效率提升：O(n log n) 比简单排序的 O(n²) 快得多
4. 代码清晰：递归结构让代码更易读
```

**评分要点：**
- 三个步骤各 3 分
- 有效性说明 6 分（至少 3 点）

### 题目 3：复杂度分析（10 分）

**参考答案：**

```
时间复杂度：
最好情况：O(n log n)
平均情况：O(n log n)
最坏情况：O(n log n)

空间复杂度：O(n)

稳定性：稳定排序

为什么需要 O(n) 的额外空间：
因为 merge 函数需要创建一个临时数组来存储合并后的结果。
合并两个长度为 n/2 的数组，需要 O(n) 的额外空间。
```

**详细解释：**

**时间复杂度推导：**
```
归并排序的递归树高度为 log n 层
每层需要合并 n 个元素，耗时 O(n)
总时间 = 层数 × 每层时间 = O(n) × O(log n) = O(n log n)

例如：n = 8
第 0 层：合并 8 个元素 → O(8)
第 1 层：合并 4+4 个元素 → O(8)
第 2 层：合并 2+2+2+2 个元素 → O(8)
第 3 层：8 个单元素 → O(1)
总共：O(8) × 3 层 = O(8 log 8)
```

**空间复杂度推导：**
```
每次合并都需要创建临时数组
最大时需要存储 n 个元素
所以空间复杂度是 O(n)
```

**评分要点：**
- 时间复杂度全对得 4 分
- 空间复杂度正确得 2 分
- 稳定性正确得 2 分
- 解释合理得 2 分

## 二、代码实践题答案

### 题目 4：实现 merge 函数（20 分）

**参考答案：**

```javascript
/**
 * 合并两个有序数组
 * @param {number[]} left - 左边的有序数组
 * @param {number[]} right - 右边的有序数组
 * @return {number[]} 合并后的有序数组
 */
function merge(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    // 双指针比较，将较小的元素放入结果数组
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i]);
            i++;
        } else {
            result.push(right[j]);
            j++;
        }
    }
    
    // 处理剩余元素
    // 注意：只有一个数组会有剩余，另一个已经空了
    return result.concat(left.slice(i)).concat(right.slice(j));
}

// 测试
console.log(merge([1, 3, 5], [2, 4, 6]));
// [1, 2, 3, 4, 5, 6] ✓

console.log(merge([], [1, 2, 3]));
// [1, 2, 3] ✓

console.log(merge([1, 3, 5], []));
// [1, 3, 5] ✓

console.log(merge([1, 3], [2]));
// [1, 2, 3] ✓
```

**执行过程演示：**
```
merge([1, 3, 5], [2, 4, 6])

初始：result=[], i=0, j=0

第 1 轮：
left[0]=1, right[0]=2
1 <= 2 → result=[1], i=1

第 2 轮：
left[1]=3, right[0]=2
3 > 2 → result=[1,2], j=1

第 3 轮：
left[1]=3, right[1]=4
3 <= 4 → result=[1,2,3], i=2

第 4 轮：
left[2]=5, right[1]=4
5 > 4 → result=[1,2,3,4], j=2

第 5 轮：
left[2]=5, right[2]=6
5 <= 6 → result=[1,2,3,4,5], i=3

退出循环（i=3 等于 left.length）

处理剩余：
left.slice(3) = []
right.slice(2) = [6]
result.concat([]).concat([6]) = [1,2,3,4,5,6] ✓
```

**评分要点：**
- 初始化正确（2 分）
- while 循环条件正确（4 分）
- 比较逻辑正确（6 分）
- 处理剩余元素正确（6 分）
- 能通过所有测试（2 分）

**常见错误：**
❌ 忘记处理剩余元素 → ✅ 使用 concat 和 slice 处理
❌ 用 < 而不是 <= → ✅ 用 <= 保证稳定性
❌ 修改了原数组 → ✅ 创建新数组，不修改输入

### 题目 5：实现完整的归并排序（25 分）

**参考答案：**

```javascript
/**
 * 归并排序
 * @param {number[]} arr - 待排序数组
 * @return {number[]} 排序后的数组
 */
function mergeSort(arr) {
    // 基准情况：空数组或单元素数组
    if (arr.length <= 1) {
        return arr;
    }
    
    // 分解：找到中点，分成两半
    const mid = Math.floor(arr.length / 2);
    const left = arr.slice(0, mid);      // 左半部分
    const right = arr.slice(mid);         // 右半部分
    
    // 递归排序 + 合并
    return merge(mergeSort(left), mergeSort(right));
}

// 测试
console.log(mergeSort([64, 34, 25, 12, 22, 11, 90]));
// [11, 12, 22, 25, 34, 64, 90] ✓

console.log(mergeSort([5, 1, 4, 2, 8]));
// [1, 2, 4, 5, 8] ✓

console.log(mergeSort([]));
// [] ✓

console.log(mergeSort([1]));
// [1] ✓

console.log(mergeSort([3, 3, 3, 3]));
// [3, 3, 3, 3] ✓
```

**执行过程演示：**
```
mergeSort([5, 1, 4, 2, 8])

第 1 层递归：
[5, 1, 4, 2, 8]
→ left = [5, 1], right = [4, 2, 8]

第 2 层递归：
左边：[5, 1]
→ left = [5], right = [1]
→ merge([5], [1]) = [1, 5]

右边：[4, 2, 8]
→ left = [4], right = [2, 8]
→ merge([4], mergeSort([2, 8]))
→ merge([4], merge([2], [8]))
→ merge([4], [2, 8]) = [2, 4, 8]

第 3 层合并：
merge([1, 5], [2, 4, 8])
→ [1, 2, 4, 5, 8] ✓
```

**评分要点：**
- 基准情况正确（4 分）
- 分解逻辑正确（6 分）
- 递归调用正确（6 分）
- 合并调用正确（5 分）
- 能通过所有测试（4 分）

**关键理解：**
```
1. 为什么要 slice？
   - 创建新的子数组，避免修改原数组
   - 保证递归的正确性

2. 为什么 stable？
   - merge 函数中用 <= 而不是 <
   - 相等时优先取左边的元素
   - 保持了相对顺序

3. 时间复杂度为什么是 O(n log n)？
   - 递归树高度：log n
   - 每层合并：O(n)
   - 总计：O(n log n)
```

### 题目 8：逆序对问题（10 分）

**参考答案：**

**问题描述：**
```
给定一个数组，计算有多少对 (i, j) 满足 i < j 且 arr[i] > arr[j]。
例如：[3, 1, 2] 的逆序对有 (3,1) 和 (3,2)，共 2 个。
```

**我的思路：**
```
在归并排序的 merge 过程中统计逆序对：

1. 当 left[i] > right[j] 时，说明 left[i] 及其后面的所有元素
   都大于 right[j]（因为 left 是有序的）

2. 此时逆序对数量增加 mid - i 个
   （mid 是左半部分的长度，i 是当前左指针位置）

3. 累加所有的逆序对数量

核心技巧：
- 在 merge 时，如果发现 right[j] < left[i]
- 那么 left[i...mid-1] 都大于 right[j]
- 一次性统计 mid - i 个逆序对
```

**代码框架：**
```javascript
let count = 0;

function mergeAndCount(left, right) {
    const result = [];
    let i = 0, j = 0;
    
    while (i < left.length && j < right.length) {
        if (left[i] <= right[j]) {
            result.push(left[i++]);
        } else {
            result.push(right[j++]);
            count += left.length - i;  // 关键！
        }
    }
    
    return result.concat(left.slice(i)).concat(right.slice(j));
}
```

**预期时间复杂度：** O(n log n)
```
利用归并排序的分治思想，
在 O(n log n) 时间内完成统计。
如果用暴力方法需要 O(n²)。
```

**评分要点：**
- 理解逆序对定义 2 分
- 提出在 merge 时统计 4 分
- 说明统计方法 2 分
- 时间复杂度正确 2 分

### 题目 9：小老师时间（10 分）

**参考模板：**

```
大家好，今天我要讲的是"归并排序"。

你们可能会问，什么是归并排序呢？

其实啊，归并排序就像整理两堆扑克牌。
假设你有两堆牌，每堆都已经按从小到大排好了。
你要把这两堆牌合并成一堆有序的牌。

怎么做呢？
你左手拿一堆，右手拿一堆，
每次比较两堆顶部的牌，把小的那张拿出来。
这样一轮下来，两堆牌就合并成一堆有序的牌了。

那如果只有一堆乱的牌怎么办呢？
简单！先把这堆牌分成两半，
每一半再继续分，直到每堆只有一张牌。
然后再按照刚才的方法，两两合并。

举个例子：
[5, 1, 4, 2, 8]
第一步：分成 [5, 1] 和 [4, 2, 8]
第二步：[5, 1] 分成 [5] 和 [1]，合并成 [1, 5]
第三步：[4, 2, 8] 分成 [4] 和 [2, 8]，[2, 8] 再分成 [2] 和 [8]
第四步：合并 [4] 和 [2, 8] 得到 [2, 4, 8]
第五步：合并 [1, 5] 和 [2, 4, 8] 得到 [1, 2, 4, 5, 8]

所以，归并排序就是：
先不断分解，直到每堆只有一个元素，
然后两两合并，最后得到完整的有序序列！

谢谢大家！
```
