## 分支 1：基本原理

```
原理
    │
    ├─ 基本思想
    │   ├─ 分为已排序和未排序
    │   └─ 每轮选最小放前面
    │
    ├─ 名称由来
    │   └─ 每次选择最小元素
    │
    └─ 执行过程
        ├─ 第一轮：最小值放位置 0
        ├─ 第二轮：第二小放位置 1
        └─ 第 n-1 轮：完成排序
```

---

## 分支 2：代码实现

```
实现
    │
    ├─ 基础版本
    │   ├─ 两层循环
    │   ├─ 外层控制已排序末尾
    │   └─ 内层找最小值
    │
    ├─ 关键步骤
    │   ├─ 假设当前位置最小
    │   ├─ 遍历找更小值
    │   └─ 交换到正确位置
    │
    └─ 注意事项
        ├─ 内层从 i+1 开始
        └─ 检查是否需要交换
```

---

## 分支 3：与冒泡排序对比

```
对比
    │
    ├─ 相同点
    │   ├─ 时间复杂度 O(n²)
    │   ├─ 空间复杂度 O(1)
    │   └─ 简单易懂
    │
    ├─ 不同点
    │   ├─ 冒泡：相邻比较，多次交换
    │   └─ 选择：找最小值，一次交换
    │
    └─ 性能差异
        ├─ 冒泡最好情况 O(n)
        └─ 选择始终 O(n²)
```

---

## 分支 4：复杂度与特性

```
复杂度与特性
    │
    ├─ 时间复杂度
    │   ├─ 最好：O(n²)
    │   ├─ 平均：O(n²)
    │   └─ 最坏：O(n²)
    │
    ├─ 空间复杂度
    │   └─ O(1) - 原地排序
    │
    ├─ 稳定性
    │   └─ 不稳定排序
    │
    └─ 适用场景
        ├─ 数据量小
        ├─ 交换代价高
        └─ 教学演示
```

---

## 🔑 核心要点

### 三个关键词
```
1️⃣ 选择最小
   每轮从未排序部分选最小

2️⃣ 一次交换
   找到后只交换一次

3️⃣ 始终 O(n²)
   任何情况下都是固定复杂度
```

### 一个中心思想
```
💡 贪心策略
每一步都选择当前最优解
```
