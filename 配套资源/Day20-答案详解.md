### 题目 2：堆排序的原理（15 分）

**参考答案：**

**基本思想：**（6 分）
```
利用大顶堆的性质：根节点总是最大值。
首先将数组构建成大顶堆，
然后将堆顶元素（最大值）与末尾元素交换，
重新调整堆，重复这个过程。
```

**两个阶段：**（6 分）

**1. 建堆阶段：**（3 分）
```
从最后一个非叶节点开始，
自底向上进行下滤操作，
将整个数组构建成大顶堆。
```

**2. 排序阶段：**（3 分）
```
将堆顶元素（最大值）与末尾元素交换，
堆的大小减 1，
对新的堆顶进行下滤操作，
重复直到堆的大小为 1。
```

**为什么用大顶堆实现升序：**（3 分）
```
大顶堆的堆顶是最大值，
每次将最大值交换到末尾，
这样从后往前就形成了升序序列。
如果用降序，则使用小顶堆。
```

**评分要点：**
- 基本思想 6 分
- 两个阶段各 3 分
- 大顶堆说明 3 分

### 题目 4：实现 heapify 函数（20 分）

**参考答案：**

```javascript
/**
 * 下滤操作（维护大顶堆性质）
 * @param {number[]} arr - 数组
 * @param {number} n - 堆的大小
 * @param {number} i - 当前节点索引
 */
function heapify(arr, n, i) {
    let largest = i;           // 假设当前节点最大
    const left = 2 * i + 1;    // 左子节点
    const right = 2 * i + 2;   // 右子节点
    
    // 如果左子节点更大
    if (left < n && arr[left] > arr[largest]) {
        largest = left;
    }
    
    // 如果右子节点更大
    if (right < n && arr[right] > arr[largest]) {
        largest = right;
    }
    
    // 如果最大值不是当前节点，交换并递归调整
    if (largest !== i) {
        [arr[i], arr[largest]] = [arr[largest], arr[i]];
        
        // 递归调整子树
        heapify(arr, n, largest);
    }
}

// 辅助函数：建堆
function buildMaxHeap(arr) {
    const n = arr.length;
    
    // 从最后一个非叶节点开始
    // 最后一个非叶节点的索引是 Math.floor(n/2) - 1
    for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
}

// 测试
const arr1 = [1, 3, 5, 7, 9, 2, 4, 6, 8, 10];
heapify(arr1, arr1.length, 0);
console.log('调整后:', arr1);

console.log('建堆后:', arr2);
// 应该是一个大顶堆，例如：[90, 80, 70, 10, 40, 50, 30]
```

### 题目 5：实现完整的堆排序（25 分）

**参考答案：**

```javascript
/**
 * 构建大顶堆
 * @param {number[]} arr - 待排序数组
 */
function buildMaxHeap(arr) {
    const n = arr.length;
    
    // 从最后一个非叶节点开始
    for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
}

/**
 * 下滤操作
 */
function heapify(arr, n, i) {
    let largest = i;
    const left = 2 * i + 1;
    const right = 2 * i + 2;
    
    if (left < n && arr[left] > arr[largest]) {
        largest = left;
    }
    
    if (right < n && arr[right] > arr[largest]) {
        largest = right;
    }
    
    if (largest !== i) {
        [arr[i], arr[largest]] = [arr[largest], arr[i]];
        heapify(arr, n, largest);
    }
}

/**
 * 堆排序
 * @param {number[]} arr - 待排序数组
 * @return {number[]} 排序后的数组
 */
function heapSort(arr) {
    const n = arr.length;
    
    // 第一步：建堆
    buildMaxHeap(arr);
    
    // 第二步：逐个提取最大值
    for (let i = n - 1; i > 0; i--) {
        // 将堆顶元素（最大值）与末尾交换
        [arr[0], arr[i]] = [arr[i], arr[0]];
        
        // 调整堆（堆大小减 1）
        heapify(arr, i, 0);
    }
    
    return arr;
}

// 测试
console.log(heapSort([64, 34, 25, 12, 22, 11, 90]));
// [11, 12, 22, 25, 34, 64, 90] ✓

console.log(heapSort([5, 1, 4, 2, 8]));
// [1, 2, 4, 5, 8] ✓

console.log(heapSort([]));
// [] ✓

console.log(heapSort([1]));
// [1] ✓

console.log(heapSort([3, 3, 3, 3]));
// [3, 3, 3, 3] ✓
```

**评分要点：**
- buildMaxHeap 正确（6 分）
- 循环边界正确（i > 0）（5 分）
- 交换操作正确（5 分）
- heapify 调用正确（i 作为堆大小）（5 分）
- 能通过所有测试（4 分）

**关键理解：**
```
1. 为什么从 Math.floor(n/2) - 1 开始？
   - 这是最后一个非叶节点的索引
   - 叶节点已经满足堆性质，不需要调整
   - 从下往上可以保证子树先调整好

2. 为什么堆大小要减 1？
   - 每次交换后，最大值已经就位
   - 不再参与后续的调整
   - 相当于"移除"了堆顶元素

3. 为什么是不稳定排序？
   - 长距离交换可能改变相等元素的顺序
   - 例如：[5, 5', 3] 建堆后可能变成 [5', 5, 3]
```

### 题目 7：堆排序的应用场景（10 分）

**参考答案：**

**适用场景：**（4 分）
```
1. 大规模数据排序（n > 1000）
2. 对最坏情况有要求的场景
3. 内存有限的场景（原地排序）
4. Top K 问题
5. 优先队列实现
```

**优点：**（3 分）
```
✓ 时间复杂度稳定，始终 O(n log n)
✓ 原地排序，空间 O(1)
✓ 适合大规模数据
✓ 实现相对简单
✓ 最坏情况性能好
```

**缺点：**（3 分）
```
✗ 不稳定排序
✗ 实际性能通常不如快速排序
✗ 建堆过程较慢
✗ 缓存命中率不如快速排序
✗ 即使数据已有序也要完整执行
```

**与归并排序、快速排序的对比：**（额外加分）
```
vs 归并排序：
- 堆排序原地，归并需要 O(n) 空间
- 归并排序稳定，堆排序不稳定
- 两者都是 O(n log n)

vs 快速排序：
- 堆排序最坏 O(n log n)，快速排序最坏 O(n²)
- 快速排序平均性能更好
- 快速排序在实际中应用更广
```

- 适用场景 4 分（至少 2 点）

### 题目 8：Top K 问题（10 分）

**参考答案：**

**问题描述：**
```
给定一个数组和一个整数 k，找到数组中前 k 大的元素。
例如：[3,2,1,5,6,4] 中前 2 大的元素是 [5, 6]。
```

**我的思路：**

**方法 1（用小顶堆）：**（5 分）
```
维护一个大小为 k 的小顶堆：
1. 遍历数组的前 k 个元素，建堆
2. 对于后续每个元素：
   - 如果大于堆顶，替换堆顶并调整
   - 否则跳过
3. 最后堆中的 k 个元素就是前 k 大

时间复杂度：O(n log k)
空间复杂度：O(k)

优点：适合海量数据，只需要 O(k) 内存
```

**方法 2（用大顶堆）：**（3 分）
```
建立整个数组的大顶堆：
1. 建堆 O(n)
2. 执行 k 次 extractMax 操作 O(k log n)
3. 得到的 k 个最大值

时间复杂度：O(n + k log n)
空间复杂度：O(1)

优点：实现简单，k 较小时效率高
```

**时间复杂度分析：**（1 分）
```
方法 1：O(n log k) - 适合 k << n
方法 2：O(n + k log n) - 适合 k 较小
```

**实际应用场景：**（1 分）
```
- 热搜榜单（Top 10 搜索词）
- 游戏排行榜（Top 100 玩家）
- 电商热销榜（Top 50 商品）
- 股票涨跌幅排行（Top 20）
```

**评分要点：**
- 方法 1 正确 5 分
- 方法 2 正确 3 分
- 复杂度分析 1 分
- 应用场景 1 分
