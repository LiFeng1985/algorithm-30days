## 一、基础概念题答案

### 题目 1：什么是选择排序？（10 分）

**参考答案：**
```
选择排序是一种简单的排序算法，
每轮从待排序的元素中选择最小的，
放到已排序部分的末尾。
就像挑选队员一样，每次选最好的。
```

**评分要点：**
- ✅ 提到"选择最小元素"（4 分）
- ✅ 有自己的比喻（3 分）
- ✅ 提到放到前面或后面（3 分）

---

### 题目 2：选择排序的原理（15 分）

**参考答案：**

**基本思想：**（6 分）
```
将数组分为两部分：
已排序部分（前面）和未排序部分（后面）。
每轮从未排序部分选择最小的元素，
放到已排序部分的末尾。
```

**为什么叫"选择"：**（4 分）
```
因为每轮都要从未排序的部分
"选择"出最小（或最大）的元素，
所以叫"选择排序"。
```

**执行过程：**（5 分）
```
第一轮：找到最小值，和位置 0 交换
第二轮：找到第二小值，和位置 1 交换
...
第 n-1 轮：完成排序
```

**评分要点：**
- 基本思想 6 分
- 命名原因 4 分
- 执行过程 5 分

### 题目 3：复杂度分析（10 分）

**参考答案：**

```
时间复杂度：
最好情况：O(n²) - 即使有序也要遍历
平均情况：O(n²)
最坏情况：O(n²) - 完全逆序

空间复杂度：O(1) - 原地排序

稳定性：不稳定 - 可能改变相等元素的相对位置
```

**评分要点：**
- 时间复杂度都是 O(n²) 得 4 分
- 空间复杂度 O(1) 得 3 分
- 稳定性正确得 3 分

**解释：**
```
为什么选择排序不稳定？
例如：[5, 8, 5, 2, 9]
第一轮会选 2，和第一个 5 交换
变成：[2, 8, 5, 5, 9]
两个 5 的相对位置改变了！
```

## 二、代码实践题答案

### 题目 4：实现基础选择排序（20 分）

**参考答案：**

```javascript
function selectionSort(arr) {
    const n = arr.length;
    
    // 外层循环控制已排序部分的末尾
    for (let i = 0; i < n - 1; i++) {
        // 假设当前位置是最小的
        let minIndex = i;
        
        // 在未排序部分找真正的最小值
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        
        // 如果找到更小的，就交换
        if (minIndex !== i) {
            [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
        }
    }
    
    return arr;
}

// 测试
console.log(selectionSort([64, 34, 25, 12, 22, 11, 90]));
// [11, 12, 22, 25, 34, 64, 90] ✓

console.log(selectionSort([5, 1, 4, 2, 8]));
// [1, 2, 4, 5, 8] ✓
```

**执行过程演示：**
```
初始：[5, 1, 4, 2, 8]

第一轮（i=0）：
假设 minIndex=0（值为 5）
j=1: 1<5 → minIndex=1
j=2: 4>1 → 不变
j=3: 2>1 → 不变
j=4: 8>1 → 不变
交换位置 0 和 1：[1, 5, 4, 2, 8]

第二轮（i=1）：
假设 minIndex=1（值为 5）
j=2: 4<5 → minIndex=2
j=3: 2<4 → minIndex=3
j=4: 8>2 → 不变
交换位置 1 和 3：[1, 2, 4, 5, 8]

第三轮（i=2）：
假设 minIndex=2（值为 4）
j=3: 5>4 → 不变
j=4: 8>4 → 不变
不需要交换：[1, 2, 4, 5, 8]

第四轮（i=3）：
假设 minIndex=3（值为 5）
j=4: 8>5 → 不变
不需要交换：[1, 2, 4, 5, 8]

最终结果：[1, 2, 4, 5, 8] ✓
```

**评分要点：**
- 外层循环正确（4 分）
- 内层循环正确（6 分）
- 找到最小值索引（4 分）
- 交换操作正确（4 分）
- 能通过测试（2 分）

**常见错误：**
❌ 内层循环从 0 开始 → ✅ 应该从 i+1 开始
❌ 忘记检查 minIndex !== i → ✅ 避免不必要的交换
❌ 比较符号搞反 → ✅ 升序用 <

### 题目 5：对比冒泡排序和选择排序（25 分）

**参考答案：**

```javascript
function compareSorts() {
    const testCases = [
        [64, 34, 25, 12, 22, 11, 90],
        [1, 2, 3, 4, 5, 6, 7],  // 已经有序
        [7, 6, 5, 4, 3, 2, 1],  // 完全逆序
        [5, 5, 5, 5, 5],         // 全部相同
        [3]                       // 单个元素
    ];
    
    for (const arr of testCases) {
        console.log('\n测试数据:', arr);
        
        // 测试冒泡排序
        const start1 = performance.now();
        bubbleSort([...arr]);
        const time1 = performance.now() - start1;
        console.log('冒泡排序耗时:', time1.toFixed(2), 'ms');
        
        // 测试选择排序
        const start2 = performance.now();
        selectionSort([...arr]);
        const time2 = performance.now() - start2;
        console.log('选择排序耗时:', time2.toFixed(2), 'ms');
        
        console.log('速度比:', (time1 / time2).toFixed(2));
    }
}

function bubbleSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        for (let j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
            }
        }
    }
    return arr;
}

function selectionSort(arr) {
    const n = arr.length;
    for (let i = 0; i < n - 1; i++) {
        let minIndex = i;
        for (let j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIndex]) {
                minIndex = j;
            }
        }
        if (minIndex !== i) {
            [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
        }
    }
    return arr;
}

compareSorts();
```

**预期结果分析：**
```
测试数据：[64, 34, 25, 12, 22, 11, 90]
冒泡排序耗时：X.XX ms
选择排序耗时：Y.YY ms
速度比：X/Y

测试数据：[1, 2, 3, 4, 5, 6, 7]（已有序）
冒泡排序：很快（优化版本提前结束）
选择排序：固定时间（必须遍历完）

结论：
✓ 冒泡排序在最好情况下更快（O(n)）
✓ 选择排序在所有情况下都是 O(n²)
✓ 选择排序交换次数更少
✓ 实际性能相差不大
```

**评分要点：**
- 测试数据全面（6 分）
- 使用 performance.now() 计时（4 分）
- 两种排序都实现（6 分）
- 输出对比结果（4 分）
- 能运行并得出结论（5 分）

### 题目 8：改进思路（10 分）

**参考答案：**

**思路 1：双向选择排序**
```
每轮同时找最小和最大值，
分别放到前面和后面。
这样轮数可以减少一半。

预期效果：
减少约一半的比较次数
```

**思路 2：记录多个最小值**
```
一次遍历找到前 k 个最小值，
然后一起放到前面。
适合部分有序的数据。

预期效果：
某些情况下减少比较次数
```

**评分要点：**
- 提出合理思路 6 分
- 说明预期效果 4 分
