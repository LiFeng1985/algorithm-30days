## 一、基础概念题答案

### 题目 1：什么是动态规划？（10 分）

**参考答案：**
```
动态规划是一种算法思想，
将复杂问题分解为重叠的子问题，
通过存储子问题的解避免重复计算，
从而高效地解决问题。
就像做数学题时记住中间结果一样。
```

**评分要点：**
- ✅ 提到"分解问题"或"子问题"（3 分）
- ✅ 有自己的比喻（3 分）
- ✅ 提到"存储"或"避免重复"（4 分）

### 题目 2：动态规划的核心概念（15 分）

**参考答案：**

**基本思想：**（6 分）
```
将一个大问题分解成若干个小问题（子问题），
先解决小问题，然后利用小问题的解来构建大问题的解。
关键是记住每个小问题的解，避免重复计算。
```

**两个关键特征：**（6 分）

**1. 重叠子问题：**（3 分）
```
在解决问题的过程中，
相同的子问题会被多次计算。
例如斐波那契数列中，计算 fib(5) 需要多次计算 fib(3)、fib(2) 等。
```

**2. 最优子结构：**（3 分）
```
问题的最优解包含其子问题的最优解。
也就是说，可以通过子问题的最优解推导出原问题的最优解。
```

**为什么 DP 比朴素递归快：**（3 分）
```
朴素递归会重复计算相同的子问题，
导致指数级的时间复杂度。
DP 通过存储子问题的解（记忆化或表格），
每个子问题只计算一次，
将时间复杂度降为多项式级。
```

**空间换时间的含义：**（额外加分）
```
使用额外的存储空间（数组、哈希表等）
来保存子问题的解，
从而避免重复计算，
大幅提高运行速度。
```

**评分要点：**
- 基本思想 6 分
- 两个特征各 3 分
- 快的原因 3 分

### 题目 3：DP 的实现方法（10 分）

**参考答案：**

**方法 1：记忆化搜索（自顶向下）**（5 分）

**优点：**（2 分）
```
✓ 思路自然，接近递归的思维方式
✓ 只计算需要的子问题
✓ 实现相对简单
```

**缺点：**（1 分）
```
✗ 有递归调用开销
✗ 可能栈溢出（问题规模大时）
```

**适用场景：**（1 分）
```
子问题不是全部需要计算，
或者问题规模不大时。
```

**方法 2：递推 DP（自底向上）**（5 分）

**优点：**（2 分）
```
✓ 没有递归开销
✓ 不会栈溢出
✓ 可以进一步优化空间
```

**缺点：**（1 分）
```
✗ 需要确定计算顺序
✗ 可能计算了不需要的子问题
```

**适用场景：**（1 分）
```
所有子问题都需要计算，
或者问题规模较大时。
```

**你会选择哪种方法：**（额外加分）
```
取决于具体情况：
- 想快速实现 → 记忆化
- 追求性能 → 递推 DP
- 问题规模大 → 递推 DP
- 子问题稀疏 → 记忆化
```

**评分要点：**
- 两种方法的优缺点各 3 分
- 适用场景各 1 分
- 选择理由合理加分

## 二、代码实践题答案

### 题目 4：斐波那契数列的不同解法（25 分）

**参考答案：**

```javascript
/**
 * 方法 1：朴素递归
 * 时间复杂度：O(2^n)
 * 空间复杂度：O(n)
 */
function fibRecursive(n) {
    if (n <= 1) {
        return n;
    }
    
    return fibRecursive(n - 1) + fibRecursive(n - 2);
}

/**
 * 方法 2：记忆化搜索
 * 时间复杂度：O(n)
 * 空间复杂度：O(n)
 */
function fibMemo(n, memo = {}) {
    if (n <= 1) {
        return n;
    }
    
    // 如果已经计算过，直接返回
    if (memo[n]) {
        return memo[n];
    }
    
    // 计算并存储
    memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
    
    return memo[n];
}

/**
 * 方法 3：递推 DP
 * 时间复杂度：O(n)
 * 空间复杂度：O(n)
 */
function fibDP(n) {
    if (n <= 1) {
        return n;
    }
    
    const dp = new Array(n + 1);
    dp[0] = 0;
    dp[1] = 1;
    
    for (let i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    
    return dp[n];
}

/**
 * 方法 4：空间优化的 DP
 * 时间复杂度：O(n)
 * 空间复杂度：O(1)
 */
function fibDPOptimized(n) {
    if (n <= 1) {
        return n;
    }
    
    let prev2 = 0;  // fib(i-2)
    let prev1 = 1;  // fib(i-1)
    
    for (let i = 2; i <= n; i++) {
        const current = prev1 + prev2;
        prev2 = prev1;
        prev1 = current;
    }
    
    return prev1;
}

// 测试
console.log('fib(10):');
console.log('朴素递归:', fibRecursive(10));      // 55 ✓
console.log('记忆化:', fibMemo(10));             // 55 ✓
console.log('递推 DP:', fibDP(10));              // 55 ✓
console.log('空间优化:', fibDPOptimized(10));    // 55 ✓

// 性能对比
console.log('\n性能对比（n=35）:');
console.time('朴素递归');
fibRecursive(35);
console.timeEnd('朴素递归');  // 约 500-1000ms

console.time('记忆化');
fibMemo(35);
console.timeEnd('记忆化');    // < 1ms

console.time('递推 DP');
fibDP(35);
console.timeEnd('递推 DP');   // < 1ms

console.time('空间优化');
fibDPOptimized(35);
console.timeEnd('空间优化');  // < 1ms
```

**四种方法的对比：**

**1. 朴素递归：**
```
优点：代码最简单，容易理解
缺点：性能极差，有大量重复计算
时间复杂度：O(2^n) - 指数级
空间复杂度：O(n) - 递归栈

执行过程：
fib(5)
├─ fib(4)
│  ├─ fib(3)
│  │  ├─ fib(2)
│  │  │  ├─ fib(1)
│  │  │  └─ fib(0)
│  │  └─ fib(1)
│  └─ fib(2)  ← 重复计算！
└─ fib(3)     ← 重复计算！
   ├─ fib(2)  ← 重复计算！
   └─ fib(1)
```

**2. 记忆化搜索：**
```
优点：避免了重复计算，性能大幅提升
缺点：仍有递归开销
时间复杂度：O(n) - 每个子问题只算一次
空间复杂度：O(n) - memo 对象 + 递归栈

优化原理：
第一次计算 fib(3) 后，存入 memo
下次需要 fib(3) 时，直接从 memo 取
```

**3. 递推 DP：**
```
优点：无递归开销，性能稳定
缺点：需要 O(n) 空间
时间复杂度：O(n)
空间复杂度：O(n)

执行过程：
dp[0] = 0
dp[1] = 1
dp[2] = dp[1] + dp[0] = 1
dp[3] = dp[2] + dp[1] = 2
dp[4] = dp[3] + dp[2] = 3
dp[5] = dp[4] + dp[3] = 5
```

**4. 空间优化的 DP：**
```
优点：性能最好，空间最优
缺点：代码稍复杂
时间复杂度：O(n)
空间复杂度：O(1)

优化技巧：
只需要保存前两个值
不需要整个数组
```

**评分要点：**
- 方法 1 正确 4 分
- 方法 2 正确 7 分
- 方法 3 正确 7 分
- 方法 4 正确 7 分

**常见错误：**
❌ 忘记边界条件 → ✅ n<=1 时返回 n
❌ 记忆化忘记存储 → ✅ 计算后要存入 memo
❌ 数组索引错误 → ✅ 注意数组大小是 n+1
❌ 空间优化更新顺序错 → ✅ 先更新 prev2 再更新 prev1

### 题目 5：爬楼梯问题（20 分）

**参考答案：**

```javascript
/**
 * 方法 1：递归（会超时）
 */
function climbStairsRecursive(n) {
    if (n <= 2) {
        return n;
    }
    
    return climbStairsRecursive(n - 1) + climbStairsRecursive(n - 2);
}

/**
 * 方法 2：记忆化搜索
 */
function climbStairsMemo(n, memo = {}) {
    if (n <= 2) {
        return n;
    }
    
    if (memo[n]) {
        return memo[n];
    }
    
    memo[n] = climbStairsMemo(n - 1, memo) + climbStairsMemo(n - 2, memo);
    
    return memo[n];
}

/**
 * 方法 3：递推 DP
 */
function climbStairsDP(n) {
    if (n <= 2) {
        return n;
    }
    
    const dp = new Array(n + 1);
    dp[0] = 1;  // 到第 0 阶有 1 种方法（不动）
    dp[1] = 1;  // 到第 1 阶有 1 种方法
    
    for (let i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    
    return dp[n];
}

/**
 * 方法 4：空间优化的 DP
 */
function climbStairsOptimized(n) {
    if (n <= 2) {
        return n;
    }
    
    let prev2 = 1;  // f(n-2)
    let prev1 = 2;  // f(n-1)
    
    for (let i = 3; i <= n; i++) {
        const current = prev1 + prev2;
        prev2 = prev1;
        prev1 = current;
    }
    
    return prev1;
}

// 测试
console.log('爬 5 阶楼梯的方法数：');
console.log(climbStairsDP(5));  // 8 ✓

console.log('爬 10 阶楼梯的方法数：');
console.log(climbStairsDP(10)); // 89 ✓

console.log('爬 20 阶楼梯的方法数：');
console.log(climbStairsOptimized(20)); // 10946 ✓
```

**解题思路：**
```
为什么这是斐波那契数列？

到第 n 阶的方法数 = 
  到第 (n-1) 阶的方法数 + 最后跨 1 阶
  到第 (n-2) 阶的方法数 + 最后跨 2 阶

所以：f(n) = f(n-1) + f(n-2)

初始条件：
f(1) = 1  （1 种方法：跨 1 阶）
f(2) = 2  （2 种方法：1+1 或 2）

验证：
f(3) = f(2) + f(1) = 2 + 1 = 3
三种方法：1+1+1, 1+2, 2+1 ✓

f(4) = f(3) + f(2) = 3 + 2 = 5
f(5) = f(4) + f(3) = 5 + 3 = 8 ✓
```

**评分要点：**
- 递归正确 3 分
- 记忆化正确 5 分
- DP 正确 7 分
- 空间优化正确 5 分

### 题目 6：最小花费爬楼梯（15 分）

**参考答案：**

```javascript
/**
 * 动态规划解法
 */
function minCostClimbingStairs(cost) {
    const n = cost.length;
    
    if (n === 0) {
        return 0;
    }
    
    if (n === 1) {
        return cost[0];
    }
    
    // dp[i] 表示到达第 i 阶的最小花费
    const dp = new Array(n + 1);
    
    // 初始条件：可以从第 0 阶或第 1 阶开始
    dp[0] = 0;
    dp[1] = 0;
    
    // 状态转移
    for (let i = 2; i <= n; i++) {
        dp[i] = Math.min(
            dp[i - 1] + cost[i - 1],  // 从第 (i-1) 阶跳 1 步
            dp[i - 2] + cost[i - 2]   // 从第 (i-2) 阶跳 2 步
        );
    }
    
    return dp[n];
}

// 空间优化的版本
function minCostClimbingStairsOptimized(cost) {
    const n = cost.length;
    
    if (n === 0) return 0;
    if (n === 1) return cost[0];
    
    let prev2 = 0;  // dp[i-2]
    let prev1 = 0;  // dp[i-1]
    
    for (let i = 2; i <= n; i++) {
        const current = Math.min(
            prev1 + cost[i - 1],
            prev2 + cost[i - 2]
        );
        prev2 = prev1;
        prev1 = current;
    }
    
    return prev1;
}

// 测试
console.log(minCostClimbingStairs([10, 15, 20]));      
// 15 ✓
// 解释：从索引 1 开始（花费 15），跳 2 步到顶部

console.log(minCostClimbingStairs([1, 100, 1, 1, 1, 90, 1, 1, 80, 1])); 
// 6 ✓
// 解释：0→2→3→4→6→7→9→顶部，总花费 1+1+1+1+1+1=6
```

**解题思路：**
```
关键点：
1. 可以从索引 0 或索引 1 开始（不花费）
2. 目标是到达楼顶（最后一个位置之后）
3. 每次可以跳 1 步或 2 步

状态定义：
dp[i] = 到达第 i 个位置的最小花费

状态转移方程：
dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2])
- 从 (i-1) 跳 1 步：花费 dp[i-1] + cost[i-1]
- 从 (i-2) 跳 2 步：花费 dp[i-2] + cost[i-2]

初始条件：
dp[0] = 0  （可以从这里开始，不花钱）
dp[1] = 0  （也可以从这里开始，不花钱）

目标：
dp[n]  （n 是 cost 的长度）
```

**执行过程演示：**
```
cost = [10, 15, 20]
n = 3

dp[0] = 0
dp[1] = 0

i = 2:
dp[2] = min(dp[1]+cost[1], dp[0]+cost[0])
      = min(0+15, 0+10)
      = min(15, 10)
      = 10

i = 3:
dp[3] = min(dp[2]+cost[2], dp[1]+cost[1])
      = min(10+20, 0+15)
      = min(30, 15)
      = 15 ✓

最终答案：15
```

**评分要点：**
- 状态定义正确 3 分
- 状态转移方程正确 5 分
- 初始条件正确 3 分
- 返回值正确 2 分
- 能通过测试 2 分

### 题目 8：算法对比总结（10 分）

**参考答案：**

**1. 时间复杂度：**（2 分）
```
朴素递归：O(2^n) - 指数级
记忆化：O(n) - 线性级
DP：O(n) - 线性级
```

**2. 空间复杂度：**（2 分）
```
朴素递归：O(n) - 递归栈
记忆化：O(n) - memo 对象 + 递归栈
DP：O(n) - 数组，可优化到 O(1)
```

**3. 实现难度：**（1 分）
```
从易到难：朴素递归 < 记忆化 < DP
```

**4. 执行效率：**（2 分）
```
从高到低：空间优化 DP > 普通 DP ≈ 记忆化 >> 朴素递归
```

**5. 调试难度：**（1 分）
```
从易到难：DP < 记忆化 < 朴素递归
（递归层次深了很难调试）
```

**总结：**（2 分）
```
1. 能用 DP 就不用朴素递归
2. 想快速实现用记忆化
3. 追求性能用递推 DP
4. 空间紧张用空间优化
```

### 题目 9：小老师时间（10 分）

**参考模板：**

```
大家好，今天我要讲的是"动态规划"。

你们可能会问，什么是动态规划呢？

其实啊，动态规划就像做作业时的草稿纸。
你做一道复杂的数学题，
会把中间步骤写在草稿纸上，
这样后面用到时直接看就行，
不用重新算一遍。

举个例子：
你要计算从 1 加到 100 的和。

笨方法：
1+2=3
3+3=6
6+4=10
...
每次都从头加起，很慢。

聪明方法（动态规划）：
记住每次的结果：
加到 1 是 1
加到 2 是 3
加到 3 是 6
...
下次要用时，直接用之前的结果，
不用再重新算。

再比如爬楼梯：
你要爬到第 10 层，
每次可以爬 1 层或 2 层。

有多少种爬法呢？

动态规划的思路：
到第 1 层：1 种方法
到第 2 层：2 种方法（1+1 或 2）
到第 3 层：到第 2 层的方法 + 到第 1 层的方法
          = 2 + 1 = 3 种
到第 4 层：到第 3 层的方法 + 到第 2 层的方法
          = 3 + 2 = 5 种
...

这样一层一层算上去，
很快就能算出到第 10 层的方法数。

所以，动态规划就是：
记住小问题的答案，
避免重复计算，
用小问题的解构建大问题的解！

谢谢大家！
```
