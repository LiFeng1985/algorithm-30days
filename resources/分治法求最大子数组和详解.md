# 分治法求最大子数组和 - 超详细图解

> 🎯 **目标：彻底理解分治法的精髓**  
> 💡 **关键：理解"跨越中点"的巧妙设计**  
> ⏱️ **时间复杂度：O(n log n)**

---

## 📖 一、问题描述

### **题目：**

给定一个整数数组，找到一个**连续子数组**，使得其元素和最大。

### **示例：**

```javascript
输入：[10, -5, 20, -15, 30]

可能的子数组：
[10]           → 和 = 10
[10, -5]       → 和 = 5
[10, -5, 20]   → 和 = 25
[10, -5, 20, -15, 30] → 和 = 40 ✅ 最大！
[-5, 20, -15]  → 和 = 0
[30]           → 和 = 30

答案：40
```

---

## 🎯 二、核心思想：分治法

### **什么是分治法？**

```
分治法 = 分解 (Divide) + 解决 (Conquer) + 合并 (Combine)

步骤：
1. 把大问题拆成小问题
2. 递归解决小问题
3. 把小问题的解合并成大问题的解
```

---

### **关键洞察：**

```
最大子数组要么在：

情况 1：完全在左半边
[10, -5, 20, -15, 30]
 ↑________↑
    左边

情况 2：完全在右半边
[10, -5, 20, -15, 30]
            ↑________↑
               右边

情况 3：跨越中点
[10, -5, 20, -15, 30]
      ↑_____________↑
         跨越中点

所以：max(左，右，跨) = 答案！
```

2. rightMax = 右半边的最大子数组和 [-15, 30]

然后返回 Math.max(leftMax, rightMax, crossMax)

mid = (0 + 2) / 2 = 1

继续递归：
1. leftMax = [10] 的最大子数组
2. rightMax = [-5, 20] 的最大子数组
3. crossMax = 跨越索引 1 的最大子数组
```

#### **计算 [10] 的最大子数组**

```
数组：[10]

      ↑
   left=right

```

#### **计算 [-5, 20] 的最大子数组**

```
数组：[-5, 20]
索引：  0    1
       ↑    ↑
      left right
      
mid = (0 + 1) / 2 = 0

继续递归：
1. leftMax = [-5] → 返回 -5
2. rightMax = [20] → 返回 20
3. crossMax = 跨越索引 0 的最大子数组

计算 crossMax：
向左：从索引 0 到 0 → -5
向右：从索引 1 到 1 → 20
crossMax = -5 + 20 = 15

返回 Math.max(-5, 20, 15) = 20

✅ rightMax = 20

#### **计算 [10, -5, 20] 的 crossMax**

```
数组：[10, -5, 20]
索引： 0    1    2
           ↑
         mid=1

向左的最大和（从 mid 向左）：
i=1: sum = -5, leftSum = -5
i=0: sum = -5+10 = 5, leftSum = 5
→ 向左最大 = 5

向右的最大和（从 mid+1 向右）：
i=2: sum = 20, rightSum = 20
→ 向右最大 = 20

crossMax = 5 + 20 = 25

返回 Math.max(10, 20, 25) = 25

✅ leftMax = 25（对于第一层来说）
```

---

### **递归计算 rightMax：[-15, 30]**

```
数组：[-15, 30]
索引：  0    1
       ↑    ↑
      left right
      
mid = (0 + 1) / 2 = 0

继续递归：
1. leftMax = [-15] → 返回 -15
2. rightMax = [30] → 返回 30
3. crossMax = 跨越索引 0 的最大子数组

计算 crossMax：
向左：从索引 0 到 0 → -15
向右：从索引 1 到 1 → 30
crossMax = -15 + 30 = 15

返回 Math.max(-15, 30, 15) = 30

✅ rightMax = 30
```

---

### **计算第一层的 crossMax**

```
数组：[10, -5, 20, -15, 30]
索引： 0    1    2    3    4
                ↑
              mid=2

向左的最大和（从 mid 向左）：
i=2: sum = 20, leftSum = 20
i=1: sum = 20+(-5) = 15, leftSum = 20（不变）
i=0: sum = 15+10 = 25, leftSum = 25
→ 向左最大 = 25

向右的最大和（从 mid+1 向右）：
i=3: sum = -15, rightSum = -15
i=4: sum = -15+30 = 15, rightSum = 15
→ 向右最大 = 15

crossMax = 25 + 15 = 40
```

### **最终结果**

```
第一层递归：

rightMax = 30  (右半边 [-15,30] 的最大子数组和)
crossMax = 40  (跨越中点的最大子数组和)

返回 Math.max(25, 30, 40) = 40 ✅

答案：40
```

## 🔍 五、crossMax 函数详解

### **为什么 crossMax 要这样计算？**

```javascript
function maxCrossingSum(nums, left, mid, right) {
    // 从中点向左的最大和
    let leftSum = -Infinity;
    let sum = 0;
    for (let i = mid; i >= left; i--) {
        sum += nums[i];
        if (sum > leftSum) leftSum = sum;
    }
    
    // 从中点向右的最大和
    let rightSum = -Infinity;
    sum = 0;
    for (let i = mid + 1; i <= right; i++) {
        sum += nums[i];
        if (sum > rightSum) rightSum = sum;
    }
    
    return leftSum + rightSum;
}
```

---

### **关键点解析：**

#### **1. 为什么要从 mid 开始向左扫描？**

```
因为跨越中点的子数组必须包含 nums[mid]！

示例：[10, -5, 20, -15, 30]
                ↑
              mid=2

跨越中点的子数组可能是：
[20]              ✓ 包含 mid
[20, -15]         ✓ 包含 mid
[-5, 20]          ✓ 包含 mid
[-5, 20, -15]     ✓ 包含 mid
[10, -5, 20]      ✓ 包含 mid
...

但不可能是：
[-15, 30]         ✗ 不包含 mid（这是右半边的事）
```

---

#### **2. 为什么要分别计算左右两边？**

```
因为跨越中点的子数组 = 左边一部分 + 右边一部分

左边部分：必须包含 mid，且和最大
右边部分：从 mid+1 开始，和最大

例如：
[10, -5, 20, -15, 30]
        ↑
      mid

左边最优：[10, -5, 20] 和 = 25
右边最优：[-15, 30] 和 = 15

组合起来：[10, -5, 20, -15, 30] 和 = 40
```

---

#### **3. 为什么用 -Infinity 初始化？**

```javascript
let leftSum = -Infinity;  // 负无穷
```

**原因：** 处理全负数的情况！

```javascript
// 如果数组全是负数：[-5, -3, -8]
// 向左累加时，和会越来越小
// 但至少要选一个元素
// 所以用 -Infinity 保证能更新

示例：[-5, -3, -8]
i=2: sum=-8, leftSum=-8
i=1: sum=-11, leftSum=-8（不变）
i=0: sum=-16, leftSum=-8（不变）

即使都是负数，也能选出最大的那个！
```

## 📈 六、时间复杂度分析

### **逐层分析：**

```
第 1 层：处理 n 个元素
        crossMax 需要 O(n) 时间
        
第 2 层：处理 2 个 n/2 的子问题
        每个 crossMax 需要 O(n/2) 时间
        总共：2 × O(n/2) = O(n)
        
第 3 层：处理 4 个 n/4 的子问题
        每个 crossMax 需要 O(n/4) 时间
        总共：4 × O(n/4) = O(n)
        
...

共有 log n 层（每次规模减半）
每层总工作量都是 O(n)

总时间复杂度：O(n) × O(log n) = O(n log n)
```

---

### **递归树分析：**

```
层数    子问题数    每个规模    该层总工作
----------------------------------------
1       1          n          O(n)
2       2          n/2        O(n)
3       4          n/4        O(n)
4       8          n/8        O(n)
...
log n   n          1          O(n)

总计：O(n) × log n = O(n log n)
```

---

### **空间复杂度：**

```
递归调用栈的深度：O(log n)
额外变量：O(1)

总空间复杂度：O(log n)
```

## 🆚 七、与其他方法对比

### **方法 1：暴力法 O(n³)**

```javascript
// 枚举所有可能的子数组
function maxSubArrayBrute(nums) {
    let max = -Infinity;
    
    for (let i = 0; i < nums.length; i++) {
        for (let j = i; j < nums.length; j++) {
            let sum = 0;
            for (let k = i; k <= j; k++) {
                sum += nums[k];
            }
            max = Math.max(max, sum);
        }
    }
    
    return max;
}

时间复杂度：O(n³) ❌
```

---

### **方法 2：动态规划 O(n)** ⭐ 最优

```javascript
// Kadane 算法
function maxSubArrayDP(nums) {
    let maxSoFar = -Infinity;
    let currentMax = 0;
    
    for (let num of nums) {
        currentMax = Math.max(num, currentMax + num);
        maxSoFar = Math.max(maxSoFar, currentMax);
    }
    
    return maxSoFar;
}

时间复杂度：O(n) ✅ 最优！
空间复杂度：O(1)
```

**Kadane 算法的核心思想：**

```
走到每个位置时，决策：
- 是继续前面的子数组？
- 还是从这里重新开始？

选择较大的那个！

示例：[10, -5, 20, -15, 30]

i=0: currentMax = max(10, 0+10) = 10
     maxSoFar = 10

i=1: currentMax = max(-5, 10-5) = 5
     maxSoFar = 10

i=2: currentMax = max(20, 5+20) = 25
     maxSoFar = 25

i=3: currentMax = max(-15, 25-15) = 10
     maxSoFar = 25

i=4: currentMax = max(30, 10+30) = 40
     maxSoFar = 40 ✅

答案：40
```

---

### **三种方法对比：**

| 方法 | 时间复杂度 | 空间复杂度 | 难度 | 推荐度 |
|------|-----------|-----------|------|--------|
| **暴力法** | O(n³) | O(1) | ⭐ 简单 | ❌ 不推荐 |
| **分治法** | O(n log n) | O(log n) | ⭐⭐⭐ 中等 | ⭐⭐ 学习用 |
| **动态规划** | O(n) | O(1) | ⭐⭐ 较难 | ⭐⭐⭐⭐⭐ 最优 |

## 💡 八、分治法的优缺点

### **优点：**

```
✅ 展示了分治思想的威力
✅ 时间复杂度 O(n log n)，比暴力法好得多
✅ 可以并行化（左右子问题独立）
✅ 易于理解和实现
✅ 是学习算法设计的经典案例
```

---

### **缺点：**

```
❌ 不是最优解（动态规划是 O(n)）
❌ 实现相对复杂
❌ 需要递归，有栈溢出风险
❌ 实际运行速度不如动态规划
```

## ✅ 十一、总结

### **核心要点：**

```
1. 分治思想：
   分解 → 解决 → 合并

2. 三种情况：
   - 最大子数组在左边
   - 最大子数组在右边
   - 最大子数组跨越中点

3. crossMax 计算：
   - 必须包含中点
   - 向左扫描找最大
   - 向右扫描找最大
   - 两者相加

4. 时间复杂度：
   O(n log n)

5. 空间复杂度：
   O(log n) - 递归栈
```

### **代码模板：**

```javascript
function maxSubArray(nums) {
    return maxSubArrayHelper(nums, 0, nums.length - 1);
}

function maxSubArrayHelper(nums, left, right) {
    if (left === right) return nums[left];
    
    const mid = Math.floor((left + right) / 2);
    
    const leftMax = maxSubArrayHelper(nums, left, mid);
    const rightMax = maxSubArrayHelper(nums, mid + 1, right);
    const crossMax = maxCrossingSum(nums, left, mid, right);
    
    return Math.max(leftMax, rightMax, crossMax);
}

function maxCrossingSum(nums, left, mid, right) {
    let leftSum = -Infinity, sum = 0;
    for (let i = mid; i >= left; i--) {
        sum += nums[i];
        if (sum > leftSum) leftSum = sum;
    }
    
    let rightSum = -Infinity;
    sum = 0;
    for (let i = mid + 1; i <= right; i++) {
        sum += nums[i];
        if (sum > rightSum) rightSum = sum;
    }
    
    return leftSum + rightSum;
}
```
