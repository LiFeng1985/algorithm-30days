## 🎪 一、核心思想：分而治之

### **生活场景：整理考试卷子**

假设你是课代表，老师抱来一堆按分数排列的试卷（从小到大），让你帮忙整理。

```
试卷堆：[78, 52, 89, 61, 95, 70, 83, 66]
```

**你会怎么做？**

#### **方法 1：一个个插队（插入排序）** ❌
- 效率低，要反复移动很多试卷

#### **方法 2：分堆法（快速排序）** ✅

```
第 1 步：随便抽一张卷子作为"基准"，比如 70 分


第 2 步：把所有比 70 小的放左边，比 70 大的放右边

   [52, 61, 66]  70  [78, 89, 95, 83]
      ↑           ↑        ↑
    小于 70     基准     大于 70


第 3 步：对左右两堆继续用同样的方法

   左边：[52, 61, 66]    右边：[78, 89, 95, 83]
   
   选 66 为基准          选 83 为基准
   [52, 61] 66 []       [78] 83 [89, 95]


第 4 步：继续分...直到每堆只有 1 张卷子


最终：[52, 61, 66, 70, 78, 83, 89, 95] ✅
```

**这就是快速排序的核心思想：分而治之！**

## 📊 二、算法原理

### **三步走策略：**

```
┌─────────────────────────────────────┐
│  1️⃣ 分解 (Divide)                  │
│     选一个基准值 (pivot)            │
│     把数组分成两部分                │
│     左边 < pivot < 右边             │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  2️⃣ 解决 (Conquer)                 │
│     递归地对左右两部分排序          │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  3️⃣ 合并 (Combine)                 │
│     不需要额外操作！                │
│     因为已经在原数组上排好了        │
└─────────────────────────────────────┘
```

### **关键概念：分区 (Partition)**

```
分区前：
[3, 1, 4, 1, 5, 9, 2, 6]
              ↑
           选这个做基准 (pivot = 6)

分区后：
[3, 1, 4, 1, 5, 2]  6  [9]
        ↑            ↑   ↑
     都小于 6      基准  都大于 6

✅ pivot 左边的数都比它小
✅ pivot 右边的数都比它大
✅ pivot 已经到达最终位置！
```

## 🎨 三、完整执行过程图解

### **待排序数组：** `[3, 1, 4, 1, 5, 9, 2, 6]`

---

### **第 1 轮：整个数组**

```
初始：[3, 1, 4, 1, 5, 9, 2, 6]
      ↑                    ↑
     low                 high
     
选 pivot = arr[high] = 6
i = low - 1 = -1
j 从 low 遍历到 high-1
```

#### **详细步骤：**

```javascript

pivot = 6
i = -1
j = 0

// j=0: arr[0]=3
3 < 6 ? ✅ 
→ i++ → i=0
→ 交换 arr[0] 和 arr[0]（自己和自己换）
→ 数组不变：[3, 1, 4, 1, 5, 9, 2, 6]

// j=1: arr[1]=1
1 < 6 ? ✅
→ i++ → i=1
→ 交换 arr[1] 和 arr[1]
→ 数组不变：[3, 1, 4, 1, 5, 9, 2, 6]

// j=2: arr[2]=4
4 < 6 ? ✅
→ i++ → i=2
→ 交换 arr[2] 和 arr[2]
→ 数组不变：[3, 1, 4, 1, 5, 9, 2, 6]

// j=3: arr[3]=1
1 < 6 ? ✅
→ i++ → i=3
→ 交换 arr[3] 和 arr[3]
→ 数组不变：[3, 1, 4, 1, 5, 9, 2, 6]

// j=4: arr[4]=5
5 < 6 ? ✅
→ i++ → i=4
→ 交换 arr[4] 和 arr[4]
→ 数组不变：[3, 1, 4, 1, 5, 9, 2, 6]

// j=5: arr[5]=9
9 < 6 ? ❌
→ 不操作，j 继续

// j=6: arr[6]=2
2 < 6 ? ✅
→ i++ → i=5
→ 交换 arr[5] 和 arr[6]
→ 数组变为：[3, 1, 4, 1, 5, 2, 9, 6]
               ↑        ↑
               i        j

// 循环结束！

// i+1 = 6, high = 7
// 交换 arr[6] 和 arr[7] → 9 和 6 交换
→ 数组变为：[3, 1, 4, 1, 5, 2, 6, 9]
                          ↑
                       pivot=6
                       
返回 pivotIndex = 6
```

#### **分区结果：**

```
[3, 1, 4, 1, 5, 2]  6  [9]
        ↑            ↑   ↑
     左子数组      基准  右子数组
   都要小于 6       ✅   都要大于 6
```

### **第 2 轮：递归处理左子数组 `[3, 1, 4, 1, 5, 2]`**

```
左子数组：[3, 1, 4, 1, 5, 2]
         ↑                 ↑
        low              high
        
选 pivot = arr[high] = 2
i = low - 1 = -1

j=0: 3 < 2 ? ❌ 不操作
j=1: 1 < 2 ? ✅ → i=0 → 交换 arr[0] 和 arr[1] → [1, 3, 4, 1, 5, 2]
j=2: 4 < 2 ? ❌ 不操作
j=3: 1 < 2 ? ✅ → i=1 → 交换 arr[1] 和 arr[3] → [1, 1, 4, 3, 5, 2]
j=4: 5 < 2 ? ❌ 不操作

循环结束，交换 arr[i+1] 和 arr[high]
i+1 = 2
交换 arr[2] 和 arr[5] → 4 和 2 交换
→ [1, 1, 2, 3, 5, 4]
           ↑
        pivot=2

返回 pivotIndex = 2
```

#### **现在的状态：**

```
[1, 1]  2  [3, 5, 4]  6  [9]
  ↑     ↑     ↑       ↑   ↑
 左    基准   右      基准 基准
```

### **第 3 轮及以后：继续递归...**

```
继续对每个子数组重复上述过程...

最终会得到：
[1, 1, 2, 3, 4, 5, 6, 9] ✅
```

## 📈 五、时间复杂度分析

### **最好情况：O(n log n)**

```
每次都完美地平分！

[n 个元素]
    ↓
[n/2] + 1 + [n/2]      ← 第 1 层：需要 n 次比较
   ↓         ↓
[n/4]+1+[n/4] ...      ← 第 2 层：需要 n/2 次比较
   ↓
...

总共需要 log n 层
每层总共需要 O(n) 次操作

总时间：O(n log n) ✅
```

**图示：**

```
        [8 个元素]
           ↓
    [4] + 1 + [4]        ← 1 层
      ↓         ↓
   [2]+1+[2] [2]+1+[2]   ← 2 层
    ↓           ↓
 [1]+1+[1] ...          ← 3 层 (log₂8 = 3)
 
树的高度 = log n
每层总工作量 = n
总时间 = n × log n
```

---

### **最坏情况：O(n²)**

```
数组已经完全有序：[1, 2, 3, 4, 5, 6, 7, 8]

每次选的 pivot 都是最大值（最后一个元素）

[1, 2, 3, 4, 5, 6, 7]  8  []    ← 第 1 次：比较 7 次
[1, 2, 3, 4, 5, 6]  7  []       ← 第 2 次：比较 6 次
[1, 2, 3, 4, 5]  6  []          ← 第 3 次：比较 5 次
...

总比较次数：7+6+5+4+3+2+1 = 28 次

对于 n 个元素：(n-1) + (n-2) + ... + 1 = n(n-1)/2 = O(n²) ❌
```

---

### **平均情况：O(n log n)**

```
大部分时候，分区不会完美但也 reasonably balanced

随机数据的情况下，期望时间复杂度是 O(n log n)
```

## ⚖️ 六、优缺点对比

### ✅ **优点：**

1. **速度快** - 平均 O(n log n)，实际应用中最快的通用排序算法
2. **原地排序** - 只需要 O(log n) 额外空间（递归栈）
3. **缓存友好** - 访问局部性好，现代 CPU 喜欢
4. **实现简单** - 代码简洁，容易理解和实现
5. **分治思想** - 典型的分治算法，学习价值高

### ❌ **缺点：**

1. **最坏情况差** - O(n²)，但可以通过随机化避免
2. **不稳定** - 相同元素的相对位置可能改变
3. **递归深度** - 大数据量可能导致栈溢出（可以用尾递归优化）

## 🎯 九、代码实现（完整版）

### **基础版本：**

```javascript
/**
 * 快速排序 - Lomuto 分区方案
 * @param {number[]} arr - 待排序数组
 * @param {number} low - 起始索引
 * @param {number} high - 结束索引
 * @returns {number[]} - 排序后的数组
 */
function quickSort(arr, low = 0, high = arr.length - 1) {
    if (low < high) {
        // 获取基准值的最终位置
        const pivotIndex = partition(arr, low, high);
        
        // 递归排序左半部分
        quickSort(arr, low, pivotIndex - 1);
        
        // 递归排序右半部分
        quickSort(arr, pivotIndex + 1, high);
    }
    return arr;
}

/**
 * 分区函数 - Lomuto 方案
 * @param {number[]} arr - 待分区数组
 * @param {number} low - 起始索引
 * @param {number} high - 结束索引
 * @returns {number} - 基准值的最终位置
 */
function partition(arr, low, high) {
    // 选择最后一个元素作为基准
    const pivot = arr[high];
    
    // i 指向小于 pivot 区域的最后一个元素
    let i = low - 1;
    
    // j 扫描整个数组
    for (let j = low; j < high; j++) {
        // 如果发现比 pivot 小的元素
        if (arr[j] < pivot) {
            i++;  // 扩展小于区域
            
            // 交换 arr[i] 和 arr[j]
            [arr[i], arr[j]] = [arr[j], arr[i]];
        }
    }
    
    // 将 pivot 放到正确的位置（小于区和大于区之间）
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
    
    // 返回 pivot 的最终位置
    return i + 1;
}

// 测试
console.log(quickSort([3, 1, 4, 1, 5, 9, 2, 6]));
// 输出：[1, 1, 2, 3, 4, 5, 6, 9] ✅
```

## 🔑 十、关键问题解答

### **Q1: 为什么选最后一个元素作为 pivot？**

**答：** 这是 Lomuto 分区方案的约定，简单且易于实现。

```
其他选择 pivot 的方法：
1. 选第一个元素
2. 选中间元素
3. 随机选一个元素（推荐！）
4. 三数取中（第一个、中间、最后一个的中位数）

选择最后一个主要是为了代码简洁，不是必须的。
```

---

### **Q2: 为什么要交换 arr[i+1] 和 arr[high]？**

**答：** 为了让 pivot 到达它的最终位置。

```
分区过程中：
- i 指向所有小于 pivot 的元素的最后一个
- i+1 就是第一个大于等于 pivot 的元素的位置
- 把 pivot（在 high 位置）换到 i+1，它就正好在中间！

示例：
[3, 1, 4, 1, 5, 2]  6  [9]
              ↑
             i=4

i+1 = 5，这个位置应该放 pivot
所以交换 arr[5] 和 arr[high]
```

---

### **Q3: 快速排序是稳定的吗？**

**答：** 不是稳定的。

```
稳定性定义：相同元素的相对位置保持不变

例子：
原始：[5a, 3, 5b, 2]  （5a 和 5b 值相同，但 a 在 b 前面）

快速排序后可能变成：
[2, 3, 5b, 5a]  ❌ 5b 跑到 5a 前面了！

原因：分区时的长距离交换会打乱相同元素的顺序

如果需要稳定排序，请使用归并排序。
```

### **Q4: 如何优化快速排序？**

**答：** 有几种常见优化方法：

#### **优化 1：随机化 pivot（最重要！）**

```javascript
function partitionRandom(arr, low, high) {
    // 随机选择一个位置
    const randomIndex = Math.floor(Math.random() * (high - low + 1)) + low;
    
    // 把它换到最后，然后用标准 partition
    [arr[randomIndex], arr[high]] = [arr[high], arr[randomIndex]];
    
    return partition(arr, low, high);
}
```

**好处：** 避免最坏情况，保证期望 O(n log n)

---

#### **优化 2：小数组用插入排序**

```javascript
function quickSortOptimized(arr, low = 0, high = arr.length - 1) {
    // 小规模数据用插入排序
    if (high - low < 10) {
        insertionSort(arr, low, high);
        return arr;
    }
    
    if (low < high) {
        const pivotIndex = partition(arr, low, high);
        quickSortOptimized(arr, low, pivotIndex - 1);
        quickSortOptimized(arr, pivotIndex + 1, high);
    }
    return arr;
}

function insertionSort(arr, low, high) {
    for (let i = low + 1; i <= high; i++) {
        let value = arr[i];
        let j = i - 1;
        while (j >= low && arr[j] > value) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = value;
    }
}
```

**好处：** 小数组用插入排序更快（常数因子更小）

---

#### **优化 3：三数取中法选 pivot**

```javascript
function medianOfThree(arr, low, high) {
    const mid = Math.floor((low + high) / 2);
    
    // 比较三个元素，返回中间值的位置
    if (arr[low] <= arr[mid] && arr[mid] <= arr[high]) return mid;
    if (arr[high] <= arr[mid] && arr[mid] <= arr[low]) return mid;
    if (arr[mid] <= arr[low] && arr[low] <= arr[high]) return low;
    if (arr[high] <= arr[low] && arr[low] <= arr[mid]) return low;
    return high;
}

function partitionMedian(arr, low, high) {
    const pivotIndex = medianOfThree(arr, low, high);
    [arr[pivotIndex], arr[high]] = [arr[high], arr[pivotIndex]];
    return partition(arr, low, high);
}
```

**好处：** pivot 更接近真正的中位数，分区更平衡

### **Q5: 快速排序的空间复杂度为什么是 O(log n)？**

**答：** 因为递归调用栈的深度。

```
最好情况（完美平衡）：
quickSort(n)
  ├── quickSort(n/2)
  │     ├── quickSort(n/4)
  │     └── quickSort(n/4)
  └── quickSort(n/2)
        ├── quickSort(n/4)
        └── quickSort(n/4)

树的高度 = log₂n
每层递归需要 O(1) 空间
总空间 = O(log n)

最坏情况（完全不平衡）：
quickSort(n)
  └── quickSort(n-1)
       └── quickSort(n-2)
            └── ...
                 └── quickSort(1)

树的高度 = n
总空间 = O(n)

平均情况：O(log n)
```

## 🔗 十三、扩展阅读

### **其他分区方案：**

1. **Hoare 分区** - 原始的快速排序分区方案，效率更高但稍复杂
2. **三路快排** - 处理大量重复元素时更高效
3. **双轴快排** - Java 7+ 的 Arrays.sort() 使用

### **相关算法：**

- 归并排序 - 同样是 O(n log n)，但是稳定的
- 堆排序 - 同样是不稳定的 O(n log n)
- 计数排序 - 特殊情况下可以 O(n)
