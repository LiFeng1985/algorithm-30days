              ↑
           只有 5 和 4 位置反了

[1, 2, 3, 4, 5, 6, 7, 9, 8]

### **插入排序的执行过程：**

```javascript
function insertionSort(arr) {
    for(let i = 1; i < arr.length; i++){
        let value = arr[i]
        let j = i - 1
        
        while(j >= 0 && arr[j] > value){
            arr[j + 1] = arr[j]
            j--
        }
        
        arr[j + 1] = value
    }
}
```

**关键点：while 循环的执行次数！**

---

### **最好情况（完全有序）：**

```javascript
数组：[1, 2, 3, 4, 5, 6, 7, 8, 9]

i=1: value=2, arr[0]=1 > 2? ❌ 
   → while 循环不执行，直接插入
   → 比较 1 次，移动 0 次

i=2: value=3, arr[1]=2 > 3? ❌
   → while 循环不执行
   → 比较 1 次，移动 0 次

...

i=8: value=9, arr[7]=8 > 9? ❌
   → while 循环不执行
   → 比较 1 次，移动 0 次

总计：
- 比较次数：n-1 次 ≈ n 次
- 移动次数：0 次
- 时间复杂度：O(n) ✅ 线性时间！
```

### **基本有序的情况（只有几个元素错位）：**

```javascript
数组：[1, 2, 3, 5, 4, 6, 7, 8, 9]
              ↑  ↑
           这两个反了

i=1: value=2, 1>2? ❌ → 比较 1 次，移动 0 次
i=2: value=3, 2>3? ❌ → 比较 1 次，移动 0 次
i=3: value=5, 3>5? ❌ → 比较 1 次，移动 0 次

i=4: value=4 🔑 关键来了！
   j=3: arr[3]=5 > 4? ✅ → 移动 5
   j=2: arr[2]=3 > 4? ❌ → 停止
   → 比较 2 次，移动 1 次
   
i=5: value=6, 5>6? ❌ → 比较 1 次，移动 0 次
i=6: value=7, 6>7? ❌ → 比较 1 次，移动 0 次
i=7: value=8, 7>8? ❌ → 比较 1 次，移动 0 次
i=8: value=9, 8>9? ❌ → 比较 1 次，移动 0 次

总计：
- 比较次数：n + 1 次 ≈ n 次
- 移动次数：1 次
- 时间复杂度：O(n) ✅ 还是线性时间！
```

---

### **如果有 k 个元素错位：**

```
假设 n 个元素中，有 k 个位置不对（k << n）

每个错位的元素：
- 最多需要移动 k 次找到位置
- 其他 n-k 个元素各比较 1 次

总操作数：
≈ (n-k) × 1 + k × k
= n - k + k²

当 k 很小时（k 是常数）：
时间复杂度 = O(n) ✅

即使 k 比较小（比如 k = √n）：
时间复杂度 = O(n) ✅
```

## 🆚 与其他排序对比

### **场景：基本有序的数组**

```javascript
数组：[1, 2, 3, 4, 5, 7, 6, 8, 9]
                  ↑  ↑
               只有这两个反了
```

#### **1. 插入排序：**

```
✅ 大部分元素已经在正确位置
✅ 每轮只需要比较 1 次
✅ 只有 6 需要往前挪一格
✅ 总共约 n 次操作
⏱️ 时间：O(n) - 超快！
```

#### **2. 快速排序：**

```
❌ 不管你是否有序，都要分区
❌ 选 pivot = 9
❌ 扫描整个数组
❌ 递归调用 log n 层
❌ 总共 n log n 次操作
⏱️ 时间：O(n log n) - 较慢
```

#### **3. 归并排序：**

```
❌ 先拆分成单个元素
❌ 再合并（虽然合并很快）
❌ 还是要走完整流程
❌ 总共 n log n 次操作
⏱️ 时间：O(n log n) - 较慢
```

#### **4. 冒泡排序：**

```
❌ 即使基本有序
❌ 也要多轮比较
❌ 优化版本可以提前结束
⏱️ 时间：O(n)~O(n²) - 看实现
```

1. 小数组通常基本有序

## 🧪 实验验证

### **代码测试：**

```javascript
function insertionSort(arr) {
    let comparisons = 0;
    let swaps = 0;
    
    for(let i = 1; i < arr.length; i++){
        let value = arr[i];
        let j = i - 1;
        
        while(j >= 0 && arr[j] > value){
            comparisons++;
            arr[j + 1] = arr[j];
            swaps++;
            j--;
        }
        if(j >= 0) comparisons++;
        
        arr[j + 1] = value;
    }
    
    console.log(`比较次数：${comparisons}, 移动次数：${swaps}`);
    return arr;
}

// 测试 1：完全有序
console.log('完全有序：');
insertionSort([1, 2, 3, 4, 5, 6, 7, 8, 9]);
// 输出：比较次数：8, 移动次数：0


// 测试 2：基本有序（1 个错位）
console.log('\n基本有序（1 个错位）：');
insertionSort([1, 2, 3, 5, 4, 6, 7, 8, 9]);
// 输出：比较次数：9, 移动次数：1


// 测试 3：基本有序（3 个错位）
console.log('\n基本有序（3 个错位）：');
insertionSort([1, 2, 6, 4, 5, 3, 7, 8, 9]);
// 输出：比较次数：12, 移动次数：5


// 测试 4：完全随机
console.log('\n完全随机：');
insertionSort([5, 2, 8, 1, 9, 3, 7, 4, 6]);
// 输出：比较次数：17, 移动次数：13


// 测试 5：完全逆序
console.log('\n完全逆序：');
insertionSort([9, 8, 7, 6, 5, 4, 3, 2, 1]);
// 输出：比较次数：36, 移动次数：36
```

## 🎯 实际应用建议

### **什么时候选择插入排序？**

```
✅ 适用场景：

1. 小规模数据（n < 50）
   - 简单，常数因子小
   - 即使随机也比快排慢不了多少

2. 基本有序的数据
   - 时间复杂度接近 O(n)
   - 比任何 O(n log n) 算法都快

3. 作为高级排序的补充
   - 快速排序、归并排序在子数组很小时
   - 切换成插入排序

4. 在线排序（边接收边排序）
   - 来一个元素就插入到正确位置
   - 天然适合插入排序
```

### **什么时候不用插入排序？**

```
❌ 不适用场景：

1. 大规模随机数据
   - O(n²) 太慢了

2. 完全逆序
   - 最坏情况，O(n²)

3. 要求稳定性且大数据
   - 虽然插入排序稳定
   - 但归并排序更适合大数据
```
