# 🎯 堆排序 - 5 分钟极简入门

## 💡 核心思想（30 秒理解）

### 什么是堆？
想象一群小朋友按身高排队：
```
       9 (最高)
      / \
     6   8
    / \ / \
   3  5 7  4
```
- **大顶堆**：每个爸爸都比儿子高（父节点 > 左右孩子）
- **小顶堆**：每个爸爸都比儿子矮（父节点 < 左右孩子）

### 堆排序的思路
1. **建堆**：把所有人排成大顶堆（最高的在顶端）
2. **交换**：让最高的出列（和最后一个人交换）
3. **调整**：剩下的人重新排成大顶堆
4. **重复**：继续让次高的出列...

## 📝 代码模板（背下来！）

### 完整实现

```javascript
function heapSort(arr) {
    let n = arr.length;
    
    // 第一步：建堆（从最后一个非叶子节点开始）
    for(let i = Math.floor(n/2) - 1; i >= 0; i--) {
        heapify(arr, n, i);
    }
    
    // 第二步：不断把堆顶换到末尾
    for(let i = n - 1; i > 0; i--) {
        [arr[0], arr[i]] = [arr[i], arr[0]];  // 交换
        heapify(arr, i, 0);  // 重新调整
    }
    return arr;
}

// heapify：维护大顶堆
function heapify(arr, n, i) {
    let largest = i;  // 假设当前最大
    let left = 2 * i + 1;   // 左孩子
    let right = 2 * i + 2;  // 右孩子
    
    // 找最大的
    if(left < n && arr[left] > arr[largest]) {
        largest = left;
    }
    if(right < n && arr[right] > arr[largest]) {
        largest = right;
    }
    
    // 如果最大的不是父节点，交换并继续调整
    if(largest !== i) {
        [arr[i], arr[largest]] = [arr[largest], arr[i]];
        heapify(arr, n, largest);
    }
}

// 测试
console.log(heapSort([4, 2, 6, 5, 8, 0, 7, 1]));
// [0, 1, 2, 4, 5, 6, 7, 8]
```

## 🔍 关键问题解答

### Q1: 为什么从 `Math.floor(n/2) - 1` 开始？

**答**：因为这是**最后一个非叶子节点**。

```
数组：[9, 6, 8, 3, 5, 7, 4]
索引： 0  1  2  3  4  5  6

完全二叉树：
       0(9)
      /    \
    1(6)   2(8)
    /  \    /  \
  3(3) 4(5) 5(7) 6(4)

非叶子节点：0, 1, 2（索引）
最后一个非叶子节点：2 = Math.floor(7/2) - 1
```

**叶子节点不需要调整**（没有孩子），所以从最后一个非叶子节点开始。

---

### Q2: 为什么要交换堆顶和末尾？

**答**：因为堆顶是最大值，交换后：
- 最大值到了正确的位置（数组末尾）
- 下次调整时就不包括它了

```
建堆后：[9, 6, 8, 3, 5, 7, 4]
        ↑              ↑
      最大值          末尾

交换后：[4, 6, 8, 3, 5, 7, 9]
                    ↑  ↑
                   已排序区域
                   
调整后：[8, 6, 7, 3, 5, 4, 9]
        ↑
      新的最大值
```

---

### Q3: heapify 为什么要递归？

**答**：因为交换后可能破坏了下面子树的堆性质。

```
交换前：
       4 ← 被换下来的
      / \
     6   8
    / \ / \
   3  5 7  4

4 比 8 小，不满足大顶堆！
需要继续向下调整...

调整后：
       8
      / \
     6   7
    / \ / \
   3  5 4  4
```

## 📊 复杂度分析

| 指标 | 值 | 说明 |
|------|-----|------|
| 时间复杂度 | O(n log n) | 建堆 O(n) + n 次调整 O(log n) |
| 空间复杂度 | O(1) | 原地排序，只需要常数空间 |
| 稳定性 | ❌ 不稳定 | 交换可能改变相同元素的相对顺序 |

## 🎯 记忆口诀

> **建堆从后往前扫**（非叶节点开始）
> **调整比较父子仨**（父、左、右）
> **交换堆顶到末尾**
> **递归调整不能忘**
