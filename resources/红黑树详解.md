## 🎨 二、什么是红黑树？

### **一句话定义：**

> **红黑树是一种自平衡的二叉搜索树，通过给节点涂色和旋转操作，保证树基本平衡。**

---

### **核心特点：**

```
1. ✅ 是二叉搜索树（左 < 根 < 右）
2. ✅ 每个节点有颜色：红色或黑色
3. ✅ 自动保持平衡（不会退化成链表）
4. ✅ 查找、插入、删除都是 O(log n)
```

---

## 🎯 三、红黑树的 5 条规则（必须记住！）

### **规则 1：每个节点要么是红色，要么是黑色**

```javascript
这个最简单！就像非黑即白。

节点结构：
{
    data: 5,
    color: 'RED',    // 或 'BLACK'
    left: ...,
    right: ...
}
```

---

### **规则 2：根节点必须是黑色**

```
    ❌ 错误：
    🔴 5（根是红色）
    
    ✅ 正确：
    ⚫ 5（根是黑色）
```

**为什么？** 为了统一规则，方便后续操作。

---

### **规则 3：叶子节点（NIL）必须是黑色**

```
注意：这里的叶子指的是空的 NIL 节点！

    ⚫ 5
   /   \
 NIL⚫  NIL⚫

实际画图时我们通常省略 NIL，但要记住它们存在！
```

---

### **规则 4：红色节点的子节点必须是黑色（不能有两个连续的红）**

```
❌ 错误：
    ⚫ 5
   /   \
 🔴 3   ⚫ 8
 /
🔴 1  ← 红色节点的父节点也是红色！违规！

✅ 正确：
    ⚫ 5
   /   \
 🔴 3   ⚫ 8
 /
⚫ 1  ← 红色节点的父节点是黑色
```

**记忆口诀：** "红不相邻" - 红色节点不能相邻

---

### **规则 5：从任一节点到其所有叶子路径，包含相同数量的黑色节点**

```
这是最难理解的！看图解：

        ⚫ 4
       /   \
     🔴 2   ⚫ 6
     / \   / \
   ⚫ 1 ⚫ 3 ⚫ 5 ⚫ 7

从根节点 4 出发：
- 往左走：4(黑) → 2(红) → 1(黑) = 2 个黑
- 往左走：4(黑) → 2(红) → 3(黑) = 2 个黑
- 往右走：4(黑) → 6(黑) → 5(黑) = 3 个黑？❌

等等！别忘了 NIL 叶子！

完整路径：
- 4 → 2 → 1 → NIL = 4(黑)+1(黑)+NIL(黑) = 3 个黑
- 4 → 2 → 3 → NIL = 4(黑)+3(黑)+NIL(黑) = 3 个黑
- 4 → 6 → 5 → NIL = 4(黑)+6(黑)+5(黑)+NIL(黑) = 4 个黑？❌

还是不对？让我重新画...

实际上 6 应该是红色才能平衡：

        ⚫ 4
       /   \
     🔴 2   🔴 6
     / \   / \
   ⚫ 1 ⚫ 3 ⚫ 5 ⚫ 7

现在每条路径都是 3 个黑色！✅
```

**记忆口诀：** "黑高相等" - 所有路径的黑色高度相等

---

## 🎨 四、为什么这些规则能保证平衡？

### **关键洞察：**

```
规则 4（红不相邻）+ 规则 5（黑高相等）
= 最长路径不超过最短路径的 2 倍！

为什么？

最短路径：全黑
⚫ → ⚫ → ⚫ → NIL = 3 个黑

最长路径：红黑相间
⚫ → 🔴 → ⚫ → 🔴 → ⚫ → NIL = 3 个黑

最长路径长度 = 6
最短路径长度 = 4

6 < 2 × 4 = 8 ✅

所以树的高度被限制在 O(log n)！
```

## 🔄 五、核心操作：变色和旋转

### **操作 1：变色（Color Flip）**

```javascript
// 把节点颜色反转
node.color = !node.color

// 通常用于：
- 红变黑
- 黑变红
```

**示例：**

```
变色前：
    ⚫ 5
   /   \
 🔴 3   🔴 7

变色后：
    ⚫ 5
   /   \
 ⚫ 3   ⚫ 7
```

---

### **操作 2：左旋（Left Rotate）**

```
左旋：把右边的节点提起来，变成新的根

    ⚫ 5              🔴 7
   /   \            /   \
  A    🔴 7   =>   ⚫ 5    C
      /   \       /   \
     B     C     A     B

5 下降成 7 的左孩子
7 上升成新的根
```

**代码实现：**

```javascript
function rotateLeft(node) {
    const newRoot = node.right;      // 7 成为新根
    node.right = newRoot.left;       // 5 的右孩子变成 B
    newRoot.left = node;             // 7 的左孩子变成 5
    return newRoot;                  // 返回新根
}
```

---

### **操作 3：右旋（Right Rotate）**

```
右旋：把左边的节点提起来，变成新的根

    🔴 7              ⚫ 5
   /   \            /   \
  ⚫ 5    C   =>   A     🔴 7
 /   \                 /   \
A     B               B     C

7 下降成 5 的右孩子
5 上升成新的根
```

**代码实现：**

```javascript
function rotateRight(node) {
    const newRoot = node.left;       // 5 成为新根
    node.left = newRoot.right;       // 7 的左孩子变成 B
    newRoot.right = node;            // 5 的右孩子变成 7
    return newRoot;                  // 返回新根
}
```

## 🎯 六、插入操作（重点！）

### **插入步骤：**

```
1. 按二叉搜索树的方式插入新节点
2. 把新节点涂成红色
3. 如果违反规则，进行调整
```

**为什么新节点是红色？**

```
如果涂黑色：
- 会改变黑色高度，违反规则 5
- 影响所有经过这个节点的路径

如果涂红色：
- 不改变黑色高度
- 只可能违反规则 4（红相邻）
- 调整起来更容易！

所以选红色！✅
```

---

### **插入后的 3 种情况（叔叔节点是关键）：**

```
插入节点 = N（新插入，红色）
父节点 = P
祖父节点 = G
叔叔节点 = U（G 的另一个孩子）
```

---

#### **情况 1：叔叔节点是红色**

```
        ⚫ G
       /   \
     🔴 P   🔴 U
     /
   🔴 N  ← 新插入

问题：P 和 N 都是红色，违反规则 4！

解决：
1. 把 P 和 U 涂黑
2. 把 G 涂红
3. 从 G 继续向上检查

结果：
        🔴 G
       /   \
     ⚫ P   ⚫ U
     /
   🔴 N

局部合法了！但 G 变红了，可能和曾祖父冲突，继续向上调整！
```

---

#### **情况 2：叔叔节点是黑色，且 N 是 P 的左孩子（LL 情况）**

```
        ⚫ G
       /   \
     🔴 P   ⚫ U
     /
   🔴 N

解决：
1. 右旋 G
2. P 涂黑，G 涂红

结果：
        🔴 P
       /   \
     🔴 N   ⚫ G
            /   \
           ...  ⚫ U

完成！✅
```

---

#### **情况 3：叔叔节点是黑色，且 N 是 P 的右孩子（LR 情况）**

```
        ⚫ G
       /   \
     🔴 P   ⚫ U
       \
       🔴 N

解决：
1. 先左旋 P（变成情况 2）
2. 再按情况 2 处理

第 1 步：左旋 P
        ⚫ G
       /   \
     🔴 N   ⚫ U
     /
   🔴 P

第 2 步：按情况 2 处理（右旋 G，变色）
        🔴 N
       /   \
     🔴 P   ⚫ G
            /   \
           ...  ⚫ U

完成！✅
```

---

### **插入算法总结：**

```javascript
function insert(node, value) {
    // 1. 普通 BST 插入
    if (node === null) {
        return new Node(value, 'RED');
    }
    
    if (value < node.data) {
        node.left = insert(node.left, value);
    } else {
        node.right = insert(node.right, value);
    }
    
    // 2. 调整（修复红黑树性质）
    return fixUp(node);
}

function fixUp(node) {
    // 情况 1：右孩子是红色，左孩子是黑色 → 左旋
    if (isRed(node.right) && !isRed(node.left)) {
        node = rotateLeft(node);
    }
    
    // 情况 2：左孩子是红色，左左孩子也是红色 → 右旋
    if (isRed(node.left) && isRed(node.left.left)) {
        node = rotateRight(node);
    }
    
    // 情况 3：左右孩子都是红色 → 翻转颜色
    if (isRed(node.left) && isRed(node.right)) {
        flipColors(node);
    }
    
    return node;
}
```

## 💻 九、完整代码实现

### **Node 类：**

```javascript
class Node {
    constructor(data, color = 'RED') {
        this.data = data;
        this.color = color;
        this.left = null;
        this.right = null;
    }
    
    isRed() {
        return this.color === 'RED';
    }
}
```

---

### **RedBlackTree 类：**

```javascript
class RedBlackTree {
    constructor() {
        this.root = null;
    }
    
    // 插入
    insert(data) {
        this.root = this._insert(this.root, data);
        this.root.color = 'BLACK';  // 根始终是黑色
    }
    
    _insert(node, data) {
        if (node === null) {
            return new Node(data, 'RED');
        }
        
        if (data < node.data) {
            node.left = this._insert(node.left, data);
        } else if (data > node.data) {
            node.right = this._insert(node.right, data);
        } else {
            return node;  // 不插入重复值
        }
        
        // 修复红黑树性质
        return this._fixUp(node);
    }
    
    _fixUp(node) {
        // 情况 1：右红左黑 → 左旋
        if (this._isRed(node.right) && !this._isRed(node.left)) {
            node = this._rotateLeft(node);
        }
        
        // 情况 2：左红左左红 → 右旋
        if (this._isRed(node.left) && this._isRed(node.left.left)) {
            node = this._rotateRight(node);
        }
        
        // 情况 3：左右都红 → 翻转颜色
        if (this._isRed(node.left) && this._isRed(node.right)) {
            this._flipColors(node);
        }
        
        return node;
    }
    
    _rotateLeft(node) {
        const newRoot = node.right;
        node.right = newRoot.left;
        newRoot.left = node;
        
        // 交换颜色
        newRoot.color = node.color;
        node.color = 'RED';
        
        return newRoot;
    }
    
    _rotateRight(node) {
        const newRoot = node.left;
        node.left = newRoot.right;
        newRoot.right = node;
        
        // 交换颜色
        newRoot.color = node.color;
        node.color = 'RED';
        
        return newRoot;
    }
    
    _flipColors(node) {
        node.color = node.isRed() ? 'BLACK' : 'RED';
        node.left.color = node.left.isRed() ? 'BLACK' : 'RED';
        node.right.color = node.right.isRed() ? 'BLACK' : 'RED';
    }
    
    _isRed(node) {
        if (node === null) return false;
        return node.isRed();
    }
    
    // 查找
    search(data) {
        return this._search(this.root, data);
    }
    
    _search(node, data) {
        if (node === null) return null;
        
        if (data === node.data) return node.data;
        if (data < node.data) return this._search(node.left, data);
        return this._search(node.right, data);
    }
    
    // 中序遍历（验证是否是 BST）
    inorder() {
        const result = [];
        this._inorder(this.root, result);
        return result;
    }
    
    _inorder(node, result) {
        if (node !== null) {
            this._inorder(node.left, result);
            result.push(node.data);
            this._inorder(node.right, result);
        }
    }
}
```

## 📊 十一、红黑树 vs AVL 树

| 特性 | 红黑树 | AVL 树 |
|------|--------|--------|
| **平衡性** | 弱平衡 | 强平衡 |
| **查找速度** | 稍慢 | 更快 |
| **插入/删除** | 更快 | 更慢 |
| **旋转次数** | 少（最多 3 次） | 多 |
| **应用场景** | 频繁增删 | 频繁查询 |
| **实现难度** | 较难 | 更难 |

**结论：** 红黑树在实际应用中更常用！

## 📚 附录：常见面试题

### **Q1: 为什么红黑树叫"红黑"树？**

```
A: 因为每个节点要么红色要么黑色
（其实叫"黑白树"也可以😄）
```

---

### **Q2: 红黑树一定平衡吗？**

```
A: 是"近似平衡"，不是严格平衡
最长路径 ≤ 2 × 最短路径
但足以保证 O(log n) 的性能
```

---

### **Q3: 新节点为什么是红色？**

```
A: 因为红色不影响黑色高度
如果新节点是黑色，会破坏规则 5
调整起来更复杂
```

---

### **Q4: 红黑树和 HashMap 的区别？**

```
A: 
- HashMap: O(1) 平均，最坏 O(n)
- 红黑树：O(log n) 稳定
```
