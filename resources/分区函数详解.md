# 分区函数详解 - 超级图解版

> 💡 **目标：让你彻底理解 partition 函数的每一行代码**  
> 🎯 **关键：理解两个指针 i 和 j 的作用**

---

## 🎪 一、核心比喻：筛豆子

### **场景：**
你有一堆混合的豆子（黄豆和绿豆），要把它们分开：

```
原始：[黄，绿，黄，黄，绿，黄，绿，绿]
                  ↑
              选这个做标准（绿色）
```

**你的做法：**

1. 左手拿一个篮子（放比绿色小的）
2. 右手依次拿起每颗豆子检查
3. 如果是黄色 → 放到左边篮子
4. 如果是绿色 → 不动
5. 最后把标准的那颗绿色放到中间

**这就是分区函数的思想！**

---

## 📊 二、代码逐行解析

### **完整代码：**

```javascript
function partition(arr, low, high) {
    const pivot = arr[high];    // 第 1 步：选基准
    let i = low - 1;            // 第 2 步：初始化指针
    
    for (let j = low; j < high; j++) {  // 第 3 步：扫描
        if (arr[j] < pivot) {           // 第 4 步：比较
            i++;                        // 第 5 步：扩展小于区
            [arr[i], arr[j]] = [arr[j], arr[i]];  // 第 6 步：交换
        }
    }
    
    [arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];  // 第 7 步：归位
    return i + 1;                       // 第 8 步：返回位置
}
```

## 🔍 四、两个指针的核心作用

### **指针 i：小于区的右边界**

```
i 永远指向：最后一个小于 pivot 的元素

┌──────────────┬─────────────┐
│  小于 pivot  │  大于等于   │
│  ... arr[i]  │  arr[i+1].. │
└──────────────┴─────────────┘
               ↑
               i
               
i 的职责：标记小于区的边界，每次发现新的小元素时，i 向右移动
```

### **指针 j：扫描器**

```
j 从左到右扫描每个元素，像安检仪一样检查：
- 如果 arr[j] < pivot → 扔到左边（通过交换）
- 如果 arr[j] >= pivot → 不管它，继续扫描

j 的职责：检查每一个元素，决定它应该去哪边
```

## 🎭 五、生动比喻：整理书架

### **场景重现：**

你要整理一排书，按高度分为"薄书"和"厚书"两类。

```
书架：[3cm, 1cm, 4cm, 1cm, 5cm, 9cm, 2cm, 6cm]
                                  ↑
                              选 6cm 做标准
```

**你的操作步骤：**

1. **左手放在书架最左边**（`i = -1`）
2. **右手从第一本开始检查**（`j = 0`）
3. **遇到薄书（< 6cm）**：
   - 左手往右移一格
   - 把这本薄书和左手位置的书交换
   - 这样薄书就被收集到左边了
4. **遇到厚书（≥ 6cm）**：
   - 不动，继续检查下一本
5. **最后**：把标准的 6cm 那本书放到中间

## 📝 六、代码含义总结

### **每一行的真实含义：**

```javascript
const pivot = arr[high];
// 👉 选一个裁判长（基准值），所有人都要和它比

let i = low - 1;
// 👉 在数组左边画一条线，线左边是"小于区"
//    一开始线在数组外面，因为还没有任何元素

for (let j = low; j < high; j++) {
// 👉 派一个检查员 j，从左到右检查每个人
    
    if (arr[j] < pivot) {
    // 👉 如果发现某人比裁判长矮
        
        i++;
        // 👉 把"小于区"的边界往右推一格
        
        [arr[i], arr[j]] = [arr[j], arr[i]];
        // 👉 让这个矮个子和边界位置的人交换
        //    这样矮个子就进入了"小于区"
    }
}

[arr[i + 1], arr[high]] = [arr[high], arr[i + 1]];
// 👉 最后，让裁判长站到分界线上
//    左边都是比他矮的，右边都是比他高的

return i + 1;
// 👉 告诉调用者：裁判长现在站的位置是 i+1
```

## 🎯 七、为什么这样设计？

### **问题：为什么不直接创建两个新数组？**

```javascript
// 方法 1：简单但费空间（不推荐）
function partitionSimple(arr, pivot) {
    let left = [];
    let right = [];
    
    for (let x of arr) {
        if (x < pivot) left.push(x);
        else right.push(x);
    }
    
    return [...left, pivot, ...right];
}
```

**缺点：**
- 需要额外的 O(n) 空间
- 创建了太多新数组

**方法 2：原地分区（快速排序的方法）**

```javascript
// ✅ 优点：
// 1. 不需要额外空间
// 2. 直接在原数组上操作
// 3. 效率高，节省内存
```

## 🔑 八、关键理解点

### **Q1: 为什么 i 从 -1 开始？**

**答：** 因为一开始"小于区"是空的！

```
i = -1 表示：
- 小于区在数组外面（还不存在）
- 随着发现小于 pivot 的元素，i 逐渐向右移动
- i 进入数组内部，小于区逐渐扩大
```

### **Q2: 为什么要交换而不是直接移动？**

**答：** 为了保持数组完整性，不丢失数据！

```
错误做法：
[3, 1, 4, 1, 5, 9, 2, 6]
          ↑     ↑
          i     j (arr[j]=2 < 6)

如果把 2 直接移到 i+1:
[3, 1, 4, 1, 5, 2, 2, 6]  ❌ 原来的 9 被覆盖了！

正确做法（交换）：
[3, 1, 4, 1, 5, 2, 9, 6]  ✅ 9 被换到了后面，不会丢失
```

### **Q3: 为什么最后要交换 arr[i+1] 和 arr[high]？**

**答：** 为了让 pivot 到达它的最终位置！

```
分区完成后：
┌──────────────┬─────────────┬────────┐
│  小于 pivot  │  大于等于   │ pivot  │
│  ... arr[i]  │  arr[i+1].. │arr[high]│
└──────────────┴─────────────┴────────┘

pivot 还在最后面，不属于任何一边
把它换到 i+1 位置，正好在中间当分界线！
```

## 💡 十、记忆口诀

> **选个基准叫 pivot，**  
> **i 在左边画条线。**  
> **j 当 scanner 逐个查，**  
> **小的换到 i 右边。**  
> **扫描完毕换基准，**  
> **左右分开任务完！**

## ✅ 总结

**partition 函数的本质：**

1. **一个裁判长**（pivot）- 作为比较的标准
2. **两个指针**（i 和 j）- i 标记边界，j 负责扫描
3. **三次交换** - 把小的往前换，最后把 pivot 放中间
4. **一个结果** - pivot 左边都小，右边都大
