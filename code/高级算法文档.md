## 🔥 重点算法详解

### 1. LRU缓存（LeetCode 146）

**问题：** 设计一个LRU（最近最少使用）缓存，要求get和put操作都是O(1)时间复杂度。

**实现原理：**
```
哈希表 + 双向链表
- 哈希表：O(1)查找节点
- 双向链表：O(1)移动节点到头部/尾部
```

**核心代码：**
```javascript
class LRUCache {
    get(key) {
        // 1. 哈希表查找
        // 2. 移动到链表头部（标记为最近使用）
    }
    
    put(key, value) {
        // 1. 如果存在，更新值并移到头部
        // 2. 如果不存在，创建新节点
        // 3. 如果超出容量，删除尾部节点（最久未使用）
    }
}
```

**应用场景：**
- Redis缓存淘汰策略
- 操作系统页面置换
- 浏览器缓存管理

**详细实现：** [lruCache.js](./lruCache.js)

### 2. 布隆过滤器

**问题：** 如何判断一个元素是否在海量数据中，且节省空间？

**实现原理：**
```
位数组 + 多个哈希函数
- 插入：将元素映射到位数组的多个位置，设为1
- 查询：检查所有映射位置是否都是1
  - 都是1 → 可能存在（有误判）
  - 有0 → 一定不存在（100%准确）
```

**参数计算：**
```javascript
// 最优位数组大小
m = -(n * ln(f)) / (ln2)²

// 最优哈希函数个数
k = (m/n) * ln2

// n: 预期元素个数
// f: 期望误判率
// m: 位数组长度
// k: 哈希函数个数
```

**应用场景：**
- Redis缓存穿透防护
- 网页爬虫URL去重
- 垃圾邮件过滤

**详细实现：** [bloomFilter.js](./bloomFilter.js)

### 3. 跳表

**问题：** 如何在链表中实现O(log n)的快速查找？

**实现原理：**
```
多层索引链表
- 底层：完整链表
- 上层：快速通道（每隔一定距离有指针）
- 查找：从最高层开始，快速跳过不需要的节点
```

**vs 红黑树：**
- ✅ 实现更简单（不需要旋转）
- ✅ 并发友好（容易实现无锁）
- ✅ 范围查询更快

**应用场景：**
- Redis有序集合（Sorted Set）
- LevelDB的MemTable

**详细实现：** [skipList.js](./skipList.js)

### 4. Dijkstra最短路径

**问题：** 如何找到从起点到所有其他节点的最短路径？

**实现原理：**
```
贪心策略 + 优先队列
1. 维护距离数组（初始无穷大，起点为0）
2. 每次取出距离最小的节点
3. 用该节点更新邻居的距离
4. 重复直到所有节点都处理完
```

**关键代码：**
```javascript
function dijkstra(graph, start) {
    const distances = {}; // 最短距离
    const pq = new PriorityQueue();
    
    while (!pq.isEmpty()) {
        const current = pq.pop();
        // 更新邻居距离
        for (let neighbor in graph[current]) {
            const newDist = distances[current] + weight;
            if (newDist < distances[neighbor]) {
                distances[neighbor] = newDist;
                pq.push(neighbor, newDist);
            }
        }
    }
}
```

**应用场景：**
- 地图导航（高德、百度）
- 网络路由（OSPF协议）
- 游戏AI寻路

**详细实现：** [dijkstra.js](./dijkstra.js)

### 5. 拓扑排序

**问题：** 如何安排有依赖关系的任务顺序？

**实现原理（Kahn算法）：**
```
1. 统计所有节点的入度
2. 将入度为0的节点加入队列
3. 依次取出节点，删除其所有出边
4. 如果有节点入度变为0，加入队列
5. 如果最终处理的节点数 < 总节点数 → 有环
```

**应用场景：**
- 课程安排（LeetCode 207/210）
- 任务调度（Jenkins、Makefile）
- npm依赖解析

**详细实现：** [topologicalSort.js](./topologicalSort.js)

### 6. 背包问题

**0-1背包（每个物品只能选一次）：**
```javascript
// 状态转移方程
dp[j] = max(dp[j], dp[j-weight[i]] + value[i])

// 注意：逆序遍历（防止重复选择）
for (let j = capacity; j >= weight[i]; j--)
```

**完全背包（每个物品可以选无限次）：**
```javascript
// 状态转移方程相同
// 注意：正序遍历（允许重复选择）
for (let j = weight[i]; j <= capacity; j++)
```

**应用场景：**
- 资源分配
- 投资决策
- 零钱兑换（LeetCode 322）

**详细实现：** [knapsack.js](./knapsack.js)

### 7. KMP字符串匹配

**问题：** 如何在文本串中快速查找模式串？

**核心思想：**
```
利用已匹配的信息，避免重复比较
- 构建next数组（部分匹配表）
- 失配时，不回溯文本串指针
- 只移动模式串指针
```

**next数组含义：**
```
next[i] = 模式串前i个字符中，最长相等前后缀的长度

例如："ABABC"
next[0] = -1
next[1] = 0  ("A"没有前后缀)
next[2] = 0  ("AB"没有相等前后缀)
next[3] = 1  ("ABA"，前缀"A"=后缀"A")
next[4] = 2  ("ABAB"，前缀"AB"=后缀"AB")
```

**时间复杂度：** O(n+m)，比暴力匹配O(n*m)快很多

**应用场景：**
- 文本编辑器查找（Ctrl+F）
- DNA序列匹配
- 入侵检测系统

**详细实现：** [kmp.js](./kmp.js)

### 8. 最小生成树

**Prim算法（从点出发）：**
```
类似Dijkstra，每次选最短的边扩展
- 适合稠密图（边多）
- 时间复杂度：O(E log V)
```

**Kruskal算法（从边出发）：**
```
1. 按边权排序
2. 从小到大选边
3. 用并查集检测环
- 适合稀疏图（边少）
- 时间复杂度：O(E log E)
```

**应用场景：**
- 网络设计（电信、电力）
- 道路建设
- 电路布线

**详细实现：** [minimumSpanningTree.js](./minimumSpanningTree.js)

## 📖 使用示例

### 快速开始

```javascript
// LRU缓存
const lru = new LRUCache(2);
lru.put(1, 1);
lru.put(2, 2);
console.log(lru.get(1)); // 1

// 布隆过滤器
const bloom = new BloomFilter(1000, 0.01);
bloom.add('apple');
console.log(bloom.mightContain('apple')); // true

// 最短路径
const graph = {
    'A': { 'B': 4, 'C': 2 },
    'B': { 'C': 1, 'D': 5 },
    'C': { 'D': 8 },
    'D': {}
};
const result = dijkstra(graph, 'A');
console.log(result.distances); // {A:0, B:4, C:2, D:9}
```

## 📈 性能对比

### LRU缓存 vs Map

| 操作 | LRU缓存 | 普通Map |
|------|---------|---------|
| get | O(1) | O(1) |
| put | O(1) | O(1) |
| 淘汰策略 | 自动LRU | 无 |
| 空间 | O(capacity) | O(n) |

### Dijkstra vs BFS

| 特性 | Dijkstra | BFS |
|------|----------|-----|
| 适用场景 | 加权图 | 无权图 |
| 时间复杂度 | O((V+E)log V) | O(V+E) |
| 数据结构 | 优先队列 | 普通队列 |

### KMP vs 暴力匹配

| 特性 | KMP | 暴力匹配 |
|------|-----|---------|
| 时间复杂度 | O(n+m) | O(n*m) |
| 文本串指针 | 不回溯 | 回溯 |
| 适用场景 | 长文本搜索 | 短文本 |
